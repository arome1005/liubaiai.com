# 七大功能区 + 设置 — 参数 / 样式 / 可实现性

> **说明**：「七大功能区」= 顶栏 `app-masthead-center` 内 **留白 · 推演 · 流光 · 问策 · 落笔 · 生辉 · 藏经**。**设置** 在右侧齿轮，独立一档。  
> **路由真源**：`src/App.tsx`。**壳与导航真源**：`src/components/AppShell.tsx`。**主题变量真源**：`src/index.css` `:root` + `:root[data-theme="dark"]`，逻辑在 `src/theme.ts`。  
> **样式列**：下列数值为 **当前实现**；你大改视觉时只需替换 token / class，不必改本文「业务参数」列。  
> **高保真参考（v2.0）**：`design/v0-ui-reference/` + `design/v0-ui-reference/docs/ui-design-specification.md`（Next 子工程，渐进迁入 `src/`）。

## 1. 留白（作品库）


| 维度               | 规格                                                                                                        |
| ---------------- | --------------------------------------------------------------------------------------------------------- |
| **路由**           | `/library`；首页 `/` 为 `HomePage`（入口，非「第八区」）                                                                 |
| **Path / Query** | 暂无 `:` 参数；未来可加 `?sort=`、`?tag=`                                                                           |
| **数据依赖**         | `listWorks` 等（`src/db/repo.ts` / `WritingStore`）；未来：`Work.tagIds`、卡片指标                                    |
| **顶栏态**          | `NavLink end to="/library"` → 在 `/library` 为 `active`                                                     |
| **子栏**           | 无（非 `/work/:id`）                                                                                          |
| **页面顶栏**         | 无（hub）                                                                                                    |
| **主内容建议结构**      | `main.app-main` → 页面根容器：列表/卡片栅格 + 顶区工具（新建、导入）；**max-width** 可按产品设 `min(72rem, 100%)` + 水平 `padding` 与全局一致 |
| **目标能力（规划）**     | 卡片 + 进度条 + 新建弹窗 + **多选标签**（§3.5）                                                                          |


**样式建议（待你装修）**


| 区域   | 建议 token 化                                                                         |
| ---- | ---------------------------------------------------------------------------------- |
| 作品卡片 | `--radius-md`（12px）、`border: 1px solid var(--border)`、`background: var(--surface)` |
| 进度条  | 高度 4～6px，`background: var(--surface-2)`，前景 `var(--accent)`                         |
| 空状态  | `color: var(--muted)`，主按钮用 `accent`                                                |


**可实现性**：✅ 列表与 CRUD 已有；卡片/标签/进度为 **纯前端 + 数据字段**，无技术壁垒。

## 2. 推演


| 维度               | 规格                                       |
| ---------------- | ---------------------------------------- |
| **路由**           | `/logic`                                 |
| **Path / Query** | 未来：`/logic/:projectId` 或 `?workId=` 绑定作品 |
| **数据依赖**         | 规划：大纲/卷纲/细纲/文策/日志/导图；当前多为占位页             |

**可实现性**：✅ UI 可一次做全；**AI 与本地藏经检索** 依赖步 3/9；导图依赖画布库（React Flow 等），均为常见技术栈。

## 9. 与「写作」相关的隐藏主路径（不在七字里）

## 10. 可实现性总表


| 功能区 | UI / 样式 | 数据与逻辑      | 外部依赖       |
| --- | ------- | ---------- | ---------- |
| 留白  | ✅ 任意重做  | ✅ 已有 Store | 无          |
| 推演  | ✅       | 🔜 纲与日志模型  | LLM、可选藏经检索 |
| 流光  | ✅       | 🔜 碎片表     | LLM        |
| 问策  | ✅       | 🔜 会话持久化策略 | LLM        |
| 落笔  | ✅       | ✅ 圣经底座     | 无          |
| 生辉  | ✅       | 🔜 装配器与草稿流 | LLM        |
| 藏经  | ✅       | ✅ 本地块      | 可选向量库（端侧）  |
| 设置  | ✅       | ◐ 部分       | 无          |


**结论**：**样式与布局没有「做不出来」的部分**，全部是标准 Web；不确定的只有 **你愿意定成什么样的视觉**（换 token 即可）。功能上 **推演/生辉/问策** 的难点在 **模型与上下文工程**，不在 CSS。
