## D. Web 端 UI 装修执行步骤（逐功能区，含关联功能）

> **用法**：每次只装修 **一个**功能区。每个功能区完成后必须：
>
> - 视觉：与 `design/v0-ui-reference/docs/ui-design-specification.md` 对应章节 **结构一一对应**
> - 功能：仍接 `src/db/repo` / `src/storage/`* 真实数据（非 mock），三态可见（空/错/加载）
> - 联动：本区列出的“关联功能验收”逐条走通
> - 工程：`npm run build` 通过；不引入明显卡顿（滚动、打开弹层、切换 Tab 不掉帧）

### D0. 全局壳层（顶栏/布局/响应式）

- **v0 对照**：`components/app-shell.tsx`（参考）＋规范 §1（视觉系统/响应式断点）
- **主应用落点**：`src/components/AppShell.tsx`、`src/components/EditorShell.tsx`、`src/index.css`
- **装修步骤（结构一一对应）**
  - **导航层级**：顶栏七模块/“更多”收纳/当前高亮与 v0 同位
  - **宽屏舒适**：非写作页内容区有 `max-width` 与居中留白；写作页双栏不挤压
  - **窄屏策略**：顶栏按钮/导航自动换行或横向滚动；写作右栏抽屉化（overlay 可点关闭）
  - **滚动策略**：主区与右栏独立滚动；弹层打开时不出现“双滚动条”
  - **键盘可用**：⌘/Ctrl+K 命令面板不抢输入框；Esc 行为与 v0 预期一致
- **关联功能验收**
  - 登录态/非登录态导航一致；深链路（`/work/:id`、`/work/:id/summary`）回退路径不乱
  - 右栏抽屉在 13 寸/14 寸/外接宽屏下均可用，不遮挡关键按钮
  - 未登录访问任意业务页（`/library`/`/logic`/`/reference` 等）会跳转到注册/登录页；登录/注册成功后落到 `/library`

### D1. 留白（作品库门户）`/library`

- **v0 对照**：规范 §2「留白 - 作品库门户」
- **主应用落点**：`src/pages/LibraryPage.tsx`、`src/pages/HomePage.tsx`、`src/components/WorkFormModal.tsx`
- **装修步骤（结构一一对应）**
  - **页眉结构**：标题/提示语/主操作（新建作品）位置与层级对齐
  - **作品卡片**：封面/统计/进度/主按钮（继续写作）/快捷入口（推演/本书锦囊）布局对齐
  - **筛选与排序**：工具条密度、搜索框、网格/列表切换（若 v0 有则对齐）
  - **新建作品弹窗**：字段顺序、必填提示、创建成功后的引导提示与 v0 对齐
  - **三态**：无作品空态卡；加载态 skeleton/文案；失败态可见
- **关联功能验收**
  - 新建作品 → 自动进入推演（或提示去推演）闭环
  - 卡片“继续写作”→ 进入写作页选中正确章节；返回作品库不丢状态

- **装修步骤（结构一一对应）**
