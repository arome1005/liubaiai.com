## 一、设计原则与视觉系统

### 1.1 核心设计原则


| 原则        | 描述                | 具体实践                  |
| --------- | ----------------- | --------------------- |
| **数据主权**  | 用户对自己的创作数据拥有完全控制权 | 本地优先存储、明确的云同步提示、随时可导出 |
| **可解释性**  | AI辅助的每一步都应该透明可查   | 显示使用的材料、提供决策依据、允许人工干预 |
| **成本可控**  | 防止用户因误操作产生高额费用    | 消耗预估、阈值确认、防误触机制       |
| **人性化流程** | 符合作者的自然创作心流       | 从构思到成稿的完整闭环、灵活的入口     |

### 2.4 使用示范案例

#### 案例1：新作者创建第一部作品

**场景**: 小明是一位新手网文作者，想要创作一部都市玄幻小说

**操作流程**:

1. 进入「留白」作品库，看到空白状态提示「开始你的创作之旅」
2. 点击「+ 新建作品」按钮
3. 在弹窗中填写：
  - 作品名称：「都市之巅峰强者」
  - 简介：「一个落魄青年意外获得神秘传承，从此踏上逆袭之路...」
  - 标签：选择 [起点风] [都市] [玄幻] [系统流] [原创]
4. 点击「创建作品」，自动跳转到「推演」模块开始大纲规划

**界面反馈**:

- 创建成功后显示引导提示：「作品创建成功！建议先在推演中规划你的故事大纲」
- 顶栏显示当前作品上下文：「都市之巅峰强者」

#### 案例2：老作者管理多部作品

**场景**: 资深作者张三有5部作品，需要快速找到正在连载的那部

**操作流程**:

1. 进入「留白」看到5张作品卡片
2. 使用状态筛选，选择「连载中」
3. 筛选后只显示2部作品
4. 通过最近更新时间排序，找到目标作品
5. 悬浮卡片，点击「继续写作」直接进入编辑器

**界面特性**:

- 每张卡片显示清晰的进度条和字数统计
- 支持拖拽排序调整作品顺序
- 右键菜单提供「归档」「删除」「导出」等操作

#### 案例3：同人作者关联原著参考

**场景**: 同人作者想创作一部基于某IP的衍生作品

**操作流程**:

1. 新建作品时选择 [同人衍生] 标签
2. 系统提示「同人创作需遵守原著版权，请确保符合相关平台规则」
3. 创建后，在「藏经」模块导入原著参考
4. 进入「推演」时可选择关联藏经中的书目进行提炼参考

**特殊说明**:

- 同人作品会显示特殊标识
- 导出时会附带版权声明模板
- AI生成会自动加载原著设定（来自推演提炼）

### 6.5 使用示范案例

#### 案例1：写作时快速查阅设定

**场景**: 写到一半忘记某个角色的具体描述

**操作流程**:

1. 在编辑器中选中角色名「福伯」
2. 右键菜单选择「查看设定」（或快捷键 Ctrl+K）
3. 侧边弹出福伯的锦囊条目
4. 快速浏览：年龄58岁、真实身份是六品高手...
5. 关闭弹窗，继续写作
6. 如需修改设定，点击「编辑」直接修改

#### 案例2：添加新的伏笔

**场景**: 写作时临时想到一个伏笔需要记录

**操作流程**:

1. 选中刚写的句子：「福伯看向枯井的眼神闪过一丝异样」
2. 右键选择「标记为伏笔」
3. 弹窗让用户填写：
  - 伏笔名称：老槐树下的秘密
  - 计划回收章节：第18章
  - 回收方式：福伯带萧然发现暗道
4. 保存后，该句子添加下划线标记
5. 在伏笔面板中可以看到新增条目

#### 案例3：使用模板快速起笔

**场景**: 要写一场打斗，不知道如何开始

**操作流程**:

1. 在写作助手面板点击「写作模板」
2. 选择「战斗节奏」模板
3. 模板插入到当前光标位置：
  ```
   【试探阶段】

   【升级阶段】

   【转折点】

   【高潮】

   【收尾】
  ```
4. 逐个填充内容
5. 完成后删除模板标记，保留正文

2. 在「自动备份提醒」选择「每周」
3. 系统会在每周日提示进行备份
4. 点击「导出全部数据」
5. 选择保存位置，下载ZIP文件
6. 文件包含所有作品、设定、大纲等数据
