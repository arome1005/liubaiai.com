# 无障碍：已知局限（§11 步 50 · Living）

> **定位**：避免与产品/验收预期冲突；**不**声称「全站 WCAG 认证」。迭代时可随页面改版更新本文。

## 键盘

- **顶栏主导航**：留白 / 推演 / 流光 / 问策 / 落笔 / 生辉 / 藏经 等链接为常规 `<a>` / 按钮，可用 **Tab** 聚焦；当前路由在对应项上可见样式区分（`.is-on` 等）。
- **命令面板**：**⌘/Ctrl+K** 打开 `GlobalCommandPalette`（在 `input` / `textarea` / `contenteditable` 内不抢键）；**↑↓** 移动、**Enter** 执行。
- **作品库**「新建作品」：可通过 Tab 聚焦到入口按钮或依赖命令面板跳转至作品库后再操作。
- **写作页**：保存/切章与编辑器快捷键以 **CodeMirror** 与 `EditorPage` 绑定为准；**未**单独承诺「全文无鼠标」。

## 可见性与焦点

- 主要按钮、链接使用既有主题色与 hover；**图标型**控件（如顶栏部分）逐步补 `title` / `aria-label`（与 `master-checklist` §B 步 50「不全站一次改完」一致）。

## CodeMirror / 编辑器

- 正文编辑区基于 **CodeMirror 6**。完整无障碍（屏幕阅读器逐字导航、复杂表格模式等）**未**单独立项；若未来替换为分块编辑器或 Tiptap，需另行验收。

## 与发布清单的关系

- 发布前可对照本页与 `docs/发布检查清单.md`；若发现「可修但非阻塞」项，记入发布说明或 `docs/bug-backlog.md` 规划级条目。
