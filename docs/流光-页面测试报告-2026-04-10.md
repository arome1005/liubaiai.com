## 已识别问题与风险

### P1 - 新建碎片标题显示 `undefined`

- **现象**：通过「快速捕捉」新增的碎片，在列表中出现 `undefined测试：...` 或类似带 `undefined` 前缀的标题行。
- **影响**：视觉显著、削弱专业感；同时可能影响搜索/可读性（用户误以为数据异常）。
- **推测原因**：标题可选字段在渲染时被直接拼接（如 `【${title}】`），未对 `undefined/空字符串` 做降级。

### P2 - 侧边栏快速捕捉“待接”按钮

- **现象**：侧边栏内联捕捉展开态的图标按钮提示为「添加标签（待接）/关联作品（待接）/添加图片（待接）」。
- **影响**：用户点击可能产生“无反馈”或误解为 bug（虽然 tooltip 已提示）。

### P3 - 集合/标签的“空项”与重复项可读性

- **现象**：集合列表可能出现重复名称、空集合计数为 0 的项仍占位（如截图中的某集合 0）。
- **影响**：侧边栏噪音增加，降低查找效率；在用户数据多时更明显。

## 优化建议（不改 UI 结构前提下）

- **建议 A（P1）**：标题显示逻辑降级：无标题时不展示标题块，或使用“（未命名）/（无标题）”。
- **建议 B（P2）**：对“待接”入口增加统一的轻提示策略（点击 toast/弹窗提示“即将上线/需要在编辑弹窗完成”），避免“点了没反应”的挫败感。
- **建议 C（P3）**：侧边栏集合列表可选提供“隐藏空集合/按最近使用排序/去重提醒”，并在名称重复时附加创建时间或自动编号。
- **建议 D（导出校验）**：导出/ZIP 完成后给一个轻提示（toast）包含导出条数与文件名，便于用户确认动作发生。

## 未接通/未验证清单

- **未接通（明确标注）**：侧边栏内联捕捉里的「标签/关联作品/图片」三个入口（UI 上标注“待接”）。
- **未验证（受限于自动化环境/避免副作用）**：
  - 导入：需要本地文件选择 + confirm。
  - 转入写作的“去写作”写入链路：会跨模块写入章节（本次仅验证弹窗与选择控件出现）。
  - 扩容结果的保存/覆盖：未点击写入动作（本次仅验证生成完成）。
