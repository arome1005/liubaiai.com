# 网文长篇写作工具 — 开发路线图（0.1～6.7 全量）

> 本文档为**唯一进度真源**：跨会话以仓库内此文件为准。  
> **平台策略**：第一期 **Web 优先**（可选 PWA）；桌面第二期（Tauri/Electron 等）复用前端，存储层换实现。  
> **数据主权**：默认**本地优先**；账号与云同步不进入第一期（除非单独立项）。

- **一步 ≈ 一个可独立验收的里程碑**；步内可再拆任务，对外汇报以「步」为单位。
- **第一期含 AI**（第 5 组）；若某阶段整组延后，在文首「当前范围」里标注。

## 第 3 组：参考库（原作资料）

### 3.1 参考库数据模型

| 项目 | 内容 |
|------|------|
| **目标** | 与创作库分离、可扩展。 |
| **交付物** | **参考作品** → 卷/章 → **块**（chunk）+ 元数据；每条有 `sourceNote`（可选）。 |
| **验收标准** | 同一库内可存多部参考作品；**与当前创作作品 ID 隔离**。 |

**实现备注**：`referenceLibrary` + **`referenceChunks`** + **`referenceTokenPostings`** + **`referenceExcerpts`** + **`referenceTags`** / **`referenceExcerptTags`**；`embeddings` 预留。正文按 `REFERENCE_CHUNK_CHAR_TARGET` 分段；与创作 `works` 隔离。参考侧为**扁平书目 + 分块**（非「卷/章」子表），与上文交付物表述略有出入，以隔离与可扩展为准。Schema **v8**，见 `docs/tech.md`。

### 3.2 参考内容导入

| 项目 | 内容 |
|------|------|
| **目标** | 批量入库。 |
| **交付物** | 文件批量导入；编码检测/失败提示；大文件分块入库。 |
| **验收标准** | 百万字级文件导入不拖死 UI（后台任务 + 进度条）。 |

**实现备注**：`.txt` 以 UTF-8 解码并依据替换字符（U+FFFD）比例提示「疑似非 UTF-8」，用户可取消导入；不做 GBK 等自动转码。大文件以分块写入 + 索引进度为主。

### 3.3 全文检索（参考库）

| 项目 | 内容 |
|------|------|
| **目标** | 关键词快速定位。 |
| **交付物** | SQLite FTS 或等价；结果列表 + **跳转高亮**（章内定位）。 |
| **验收标准** | 检索响应在可接受时间内（目标值写进 `docs/perf.md`）。 |

### 3.4 摘录与笔记

| 项目 | 内容 |
|------|------|
| **目标** | 从参考正文划选沉淀为卡片。 |
| **交付物** | 划选保存摘录；**链回参考库位置**；用户笔记字段。 |
| **验收标准** | 点击摘录可跳回原文位置（或近似块）。 |

**实现备注**：侧栏摘录 **「跳转到原文」**；`/reference?ref=&ord=&hs=&he=` 深链供编辑器「在参考库打开」使用。

### 3.5 标签与筛选

| 项目 | 内容 |
|------|------|
| **目标** | 按人物/事件/设定浏览。 |
| **交付物** | 标签 CRUD；按标签过滤摘录列表。 |
| **验收标准** | 单条摘录可多标签；筛选正确。 |

### 3.6 与创作侧弱关联（可选）

| 项目 | 内容 |
|------|------|
| **目标** | 先「人工链」，不做自动推理。 |
| **交付物** | 给自己的某一章关联「参考某书某章」链接；侧栏展示链接。 |
| **验收标准** | 可从创作章跳到参考章（或展示元数据）。 |
| **可砍** | 合并进 3.4：摘录时可选「关联我书某章」，无单独页面。 |

**实现备注**：摘录关联创作章后，编辑器侧栏 **「本章关联参考」** 与便签列表 **「在参考库打开」**；参考正文按 **块 ordinal** 定位（无单独「参考卷/章」表）。

### 3.7 参考库性能与大文本

| 项目 | 内容 |
|------|------|
| **目标** | 数百万字可管理。 |
| **交付物** | 分块存储；索引重建；**禁止一次加载全书到内存**的代码路径审查。 |
| **验收标准** | 压力场景说明 + 抽样通过（打开章、滚动、检索）。 |

### 3.8 与写作进度游标联动（默认只搜「进度前」）

| 项目 | 内容 |
|------|------|
| **目标** | 避免参考检索把后文信息带进当前创作心智（可选但强烈推荐）。 |
| **交付物** | 若适用：参考库也支持「仅截至某进度」过滤（或明确不适用并写原因）。 |
| **验收标准** | 行为与 2.6 文档一致；用户可关。 |

**实现备注**：**摘录列表**支持「仅进度前」过滤（与 2.6 一致）；**参考全文检索**不按写作进度裁块，原因见参考库页说明。

## 第 5 组：AI 辅助（第一期纳入）

### 5.1 API 与密钥管理

| 项目 | 内容 |
|------|------|
| **目标** | 安全调用模型。 |
| **交付物** | 环境变量或浏览器安全存储策略；模型切换；错误与用量提示（若 API 提供）。 |
| **验收标准** | 密钥不出现在前端构建产物；文档说明。 |

### 5.2 按章上下文装配

| 项目 | 内容 |
|------|------|
| **目标** | 生成不裸奔。 |
| **交付物** | 自动拼接：**本章细纲**、人物卡摘要、勾选的世界观条目；**手工勾选为主**，RAG 增强为后话。 |
| **验收标准** | 预览装配结果再发送；用户可见将注入哪些块。 |

### 5.3 风格卡 + 调性锁（Tone lock）

| 项目 | 内容 |
|------|------|
| **目标** | 抑制文风漂移与「小白化」。 |
| **交付物** | **全书级**：人称、严肃度、是否玩梗、粗口程度等；**文风锚**（短样例）；**禁止词/套话表**；每次请求 **自动注入**，用户不可每次手抄。 |
| **验收标准** | 同书连续多章生成，调性字段在请求体中可追溯（开发态可 log 脱敏）。 |

### 5.4 生成与 diff 工作流

| 项目 | 内容 |
|------|------|
| **目标** | 不直接覆盖心血。 |
| **交付物** | 生成进 **侧边草稿区**；diff/合并；确认后写入正文。 |
| **验收标准** | 无「一键替换全书」无确认路径。 |

### 5.5 RAG v1（检索增强）

| 项目 | 内容 |
|------|------|
| **目标** | 百万字库可引用而不塞全文。 |
| **交付物** | 对**参考库 + 设定库**做检索；**top-k 限制**；**默认尊重写作进度游标**（只检索游标前正文，若索引了创作稿）。 |
| **验收标准** | 单次注入总字数上限；注入片段在 UI 列出。 |
| **分期** | 可先 **关键词/BM25**，向量后补；与 3.3 技术选型对齐。 |

### 5.6 摘要与记忆

| 项目 | 内容 |
|------|------|
| **目标** | 减轻窗口压力。 |
| **交付物** | 章摘要、卷摘要（可人工编辑）；参与装配的勾选。 |
| **验收标准** | 摘要不自动覆盖人工修改（需版本或编辑锁策略）。 |

### 5.7 流式、取消、重试与缓存

| 项目 | 内容 |
|------|------|
| **目标** | 体验与成本可控。 |
| **交付物** | 流式输出；取消；失败重试；可选本地缓存最近结果。 |
| **验收标准** | 中断不产生半篇脏数据进正文（未确认不入库）。 |

### 5.8 安全、隐私与合规提示

| 项目 | 内容 |
|------|------|
| **目标** | 用户知情同意。 |
| **交付物** | 默认 **不全文上传** 的可配置项；首次开启 AI 时的说明；**对话/ prompt 是否落库** 默认关或显式开启。 |
| **验收标准** | 隐私说明页或弹窗可查；与第 6 组协议一致。 |

### 5.9 质量护栏（生成后）

| 项目 | 内容 |
|------|------|
| **目标** | 术语与人名初筛。 |
| **交付物** | 生成结果跑一轮 **术语表/人名** 提示（规则即可）。 |
| **验收标准** | 与 4.6 共用词表或导入关系。 |

### 5.10 调性漂移检测（可选 / Should）

| 项目 | 内容 |
|------|------|
| **目标** | 长篇后期文风跑偏可感知。 |
| **交付物** | 轻量方案：与标杆段 embedding 距离提示，或禁用句式计数。 |
| **验收标准** | 仅提示不阻断亦可。 |
| **可砍** | 第二期。 |

### 5.11 批量操作与成本上限（可选）

| 项目 | 内容 |
|------|------|
| **目标** | 防止误操作烧额度。 |
| **交付物** | 单次会话 token 上限、批量生成前二次确认。 |
| **验收标准** | 超限有清晰错误。 |
