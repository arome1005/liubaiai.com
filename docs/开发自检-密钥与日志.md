# 开发自检：密钥与日志（§11 步 2）

> **目的**：满足清单「仓库与日志无明文 Key（流程自检）」——合入前人工过一遍即可，非 CI 门禁。

## 仓库

- **勿**将含真实 API Key、Refresh Token 的 `.env` 提交到 Git。根目录 `.gitignore` 已包含 `backend/.env`；团队私有密钥仅放本地或密钥管理器。
- 合并前可抽查：`git diff` 中是否出现疑似 `sk-`、`AIza` 等厂商密钥形态（仅作人工扫一眼）。

## 运行时日志

- 业务代码**不应** `console.log` 完整请求头、`apiKey` 原文或用户正文。若需调试，用截断或脱敏（例如仅打印「已配置 Key」布尔值）。

## 构建产物

- `npm run build` 产物不含用户密钥；密钥仅存浏览器 `localStorage`（见 `docs/技术说明.md`）。
