### 3.4 产品定案：藏经 × 推演 × 生辉（仿写管线）

> 本节吸收 **2026-04** 你的产品说明：**生辉 = 仿写**；**藏经** 为 **用户本地书库**；**推演** 负责 **从构思到纲、导图、文策与日志**；**正文生成** 只在纲定稿之后由 **生辉** 执行。

#### 3.4.1 藏经（本地书库）

- **定位**：用户自管的 **参考资料库**（原著、优秀同人等），**正文与分块索引仅存本机**（IndexedDB），**不上传到业务云端存储**（与当前 Hybrid 策略一致：写作/本书锦囊可云，**藏经不云**）。
- **用途（面向仿写/同人）**：材料进入藏经后，供 **推演** 阶段做 **关联与提炼**，不用于「整本外发洗稿」。

#### 3.4.2 推演（大纲 → 卷纲 → 细纲 → 导图 → 文策）

- **输入**：用户 **想法/构思**；可选 **关联藏经** 中的书目（多选）。
- **藏经关联时的处理原则**：对关联书目做 **提炼参考素材**（结构向：人物关系、冲突点、节奏、设定要点等），用于 **辅助生成大纲/卷纲/文策**；**非原文拼接、非洗稿**；产品文案与协议中需 **明确禁止把他人正文变相复述为「自己的正文」**。
- **输出层次**：
  - **大纲**（全书向）
  - **卷纲**（按卷）
  - **文策**（创作策略说明，可与大纲并列迭代）
  - **细纲**：在 **卷纲** 之下生成 **更细的章节级纲要**（支持卷纲 → 细纲的级联生成）
- **思维导图模式**：大纲稳定后，支持生成 **可视化导图**（任务线、地图/地点、NPC、反派、各卷章 **大致事件** 等子模式）；支持用户 **手改**。
- **迭代与对话**：推演内支持 **AI 对话框**，按用户意图 **改纲**；**修改方向与结论** 写入 **文策日志**（时间序）；用户在 **文策区** 提问时，可 **检索/调取** 该日志作为依据。
- **定稿**：用户确认 **推演完成**（大纲/卷纲/细纲与文策一致可交付）后，才进入 **生辉**。
- **作品标签**：生成各层纲与导图时，装配器 **后台注入** 本书 **平台/流派/同人** 标签对应的写作侧写（见 **§3.5**）。

#### 3.4.3 生辉（仿写）

- **定义**：**根据已定稿的大纲 / 卷纲 / 细纲**（及文策约束）生成 **对应正文**；不是脱离纲的「泛聊天生成」；**后台**附带本书 **标签 profile**（§3.5），**不在正文区展示**内部技法全文。
- **与藏经关系**：**不直接「照抄藏经原文」**；藏经只在 **推演阶段** 已通过 **提炼** 进入纲与文策；生辉侧主要消费 **纲 + 文策 + 本书锦囊（本作设定）**。
- **与落笔**：本作 **人物/世界观/术语** 仍以 **本书锦囊（落笔）** 为准；推演/生辉请求需能 **挂载当前作品本书锦囊**。

#### 3.4.4 本地藏经如何「参与 AI」而不上云存书


| 方式               | 说明                                                                                        |
| ---------------- | ----------------------------------------------------------------------------------------- |
| **端侧检索 + 端侧摘要**  | 浏览器内对藏经块做检索/摘要，仅将 **短结构化要点** 拼进请求；**原著全文不出库、不进云存储**。                                      |
| **用户授权的 API 调用** | 若实现为把 **节选或摘要** 发往用户自配模型：需在 UI **单独说明范围**，默认 **最小必要**；**藏经正文不上云存储** 与「请求里带摘要」是两类风险，需分别告知。 |
| **合规提示**         | 同人文/二创需在协议与界面提示 **版权与平台规则自负**；功能设计以 **提炼与原创纲** 为主，避免产品形态鼓励侵权。                             |

### 3.5 作品标签：平台/流派 + 同人「大神写作模式」（后台注入）

> 对标 **起点 / 番茄 / 七猫** 等站的 **作品标签心智**：用户在 **留白（作品）** 上为每书选择 **可修改** 的标签；**推演 / 生辉** 在生成 **大纲、卷纲、细纲、正文** 时，在 **装配器后台** 自动挂载 **对应类型的写作侧写**（节奏、结构习惯、爽点/悬念密度、叙事视角偏好等 **抽象技法**），**不向前端展示**「具体调用了哪条内部规则全文」——用户可见的仍是 **自己选的标签名称**（如「玄幻」「同人衍生」），**不在 UI 上展示**「辰东模式」「番茄风」等 **可识别具体作家的营销式文案**（避免误导与合规风险）；**内部**用 **profile id → 技法描述块** 注入模型上下文。

#### 3.5.1 标签与数据

- **标签来源**：预设 **平台向 / 流派向** 标签库（可多选），支持后续扩展；用户 **可随时在留白修改** 本书标签。
- **持久化**：挂在 **作品（Work）** 或独立 `work_tags` 关联表；导入/备份需包含该字段。
- **生效范围**：标签变更后 **新生成的** 推演/生辉请求使用新 profile；已生成纲/正文是否 **重跑** 由用户手动触发（避免静默改写）。

#### 3.5.2 后台匹配逻辑（用户不可见细节）

- **流派类**（如玄幻、都市、无限流）：匹配 **该类型的通用写作特点**（开篇钩子、升级节奏、章末断点等 **结构化提示**，存于内置配置或 JSON，**不进 UI 长文**）。
- **同人衍生**：若用户勾选，可额外启用 **「大神写作模式」** 内部档位：对应 **脱敏后的技法包**（例如「宏大开篇、群像铺垫、长线伏笔」等 **与具体作家解耦** 的描述）；**禁止**在请求里写「完全模仿某某作家」或粘贴作家原文；若需 **用户可感知的命名**（如「厚重史诗风」），仅用 **泛化风格名**，**不出现真实作家姓名**（合规：不暗示官方授权、不侵犯姓名/商誉）。

#### 3.5.3 与装配器、落笔的关系

- **装配器**：每次推演/生辉请求 **自动合并**：`作品标签 → 内部 profile 文本` + 纲/文策/本书锦囊/风格卡；**profile 段落对用户默认折叠**（与「本次使用材料」一致时可只显示「已应用作品标签：玄幻、同人衍生」一行，**不展开**内部技法全文）。
- **落笔**：**本书锦囊与风格卡** 仍优先于 **泛化标签**；冲突时以 **用户显式填写** 为准。

#### 3.5.4 合规

- 用户协议中说明：标签与内部 profile 仅为 **辅助生成**，**不保证**与任何平台签约作家或作品一致；**二创/同人** 版权与平台规则由用户自负。

### 4.2 分导航说明（摘要）

- **留白**：列表在 `/library`；缺卡片/进度条/新建弹窗；**作品标签**（平台/流派/同人衍生等，多选可编辑，见 **§3.5**）；导入 txt/md/docx 已有（2.11）。⭐ **默认落地页**见第七章。
- **推演**：占位；目标升级为 **§3.4 全链路**（纲→导图→文策日志→对话框改纲），并 **关联藏经（提炼，非洗稿）**。
- **流光 / 问策**：占位；与 **推演内对话** 边界需避免重复（问策可偏开放式拆书/策略，推演内对话偏 **改纲与文策**）。
- **落笔**：本书锦囊与风格卡数据 ✅；本作设定为 **生辉** 硬约束；另需 **提示词/模板/笔感/词典** 与生成链路对接。
- **生辉**：**仿写**——仅消费 **推演定稿的纲 + 文策 + 本书锦囊**；见 **§3.4**。
- **藏经**：**本地-only 存储**；缺 PDF、语义检索、**「未上传云端」** 强提示；与推演 **书目关联 + 提炼** 待实现。
- **设置**：缺多厂商 Key；⭐ **用户自定义 token/档位阈值 + 超阈值强制验证**；可选 **进阶防误触**（数字确认/长按/冷却，**§5.3.2**）；成本说明、API 失败态、隐私/虚构声明。

#### 5.3.1 助手补充（与「消耗控制」配套，可选采纳）


| 项             | 说明                                                                      |
| ------------- | ----------------------------------------------------------------------- |
| **软顶 vs 硬顶**  | **软顶**：超阈值先确认，用户仍可继续；**硬顶**：超过 **绝对上限** 直接禁止发请求，须改设置（防误触烧穿额度）。          |
| **会话 / 日累计**  | 除单次外，增加「**本会话** / **今日** 累计超过 M 则锁定或二次确认」，适合长时推演。                       |
| **事后对账**      | 若 API 返回 `usage`（input/output tokens），在请求结束后展示 **实际消耗**，与事前粗估对照，逐步校准文案。 |
| **无 usage 时** | 仅能用字符数、上下文档位的 **粗估**，界面标明 **「非计费凭证」**，避免与账单争议。                          |
| **分功能默认**     | **推演（纲/导图）**、**生辉（按章仿写）**、**流光（五连扩写）** 可设 **不同默认门槛**（同一套设置里分栏）。         |


#### 5.3.2 进阶防误触（超阈值确认，可选档位）


| 项          | 说明                                                  |
| ---------- | --------------------------------------------------- |
| **输入数字确认** | 超阈值或高危操作时，模态内要求输入 **当日动态码或屏幕展示的两位数**，误触成本高于纯点「确定」。  |
| **长按确认**   | 对 **整卷仿写、批量推演** 等：需 **长按按钮 ≥2 秒** 才触发请求（与模态二选一或叠加）。 |
| **冷却时间**   | 同一高危操作 **N 秒内不可连发**（防连点）。                           |
| **设置入口**   | 与 **§5.3「强制验证」** 同级：用户可选 **关闭 / 仅高危 / 全部超阈值启用**。    |
