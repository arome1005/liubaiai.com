## 1. 你当前目标的产品定义（统一口径）

你要的不是“复刻某本书内容”，而是：

1. 作品构思（你输入/粘贴）决定故事骨架；
2. 关联藏经只提供风格、节奏、类型惯例；
3. 输出要“像这一类优秀文本”，而不是“像某一段原文”。

建议在 UI 与提示词统一使用这句规则：

- **构思优先，参考侧重表达与结构，不复述参考书具体情节链。**

- 提供「一键精简注入（摘要版）」：
  - 保留命中片段核心语义（人名/冲突点/语气特征）；

- 直接注入：保留原片段（受长度上限约束）；

- `<= 400`：正常提示；
- `401~1200`：黄色提醒“上下文较重”；

## 7. 风险与控制

### 风险 1：结果“太像某一本”

- 控制：默认摘要优先、节选限量、构思优先 hard rule、情节复述禁令。

### 风险 2：结果“不像用户期待”

- 控制：作用维度明确化 + 模式预设 + 生成前摘要可校验。

### 风险 3：配置太复杂

- 控制：P0 只保留必要项（角色/维度/范围），高级项折叠到 P1。

### 风险 4：注入文本过长导致成本抖动

- 控制：注入前 token 预估、阈值分级提醒、默认推荐摘要注入。

### 风险 5：策略只在部分生成步骤生效

- 控制：把“是否启用参考策略”抽为统一函数，在各生成入口共用同一判定与装配逻辑。
