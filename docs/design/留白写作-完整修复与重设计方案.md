### P0-02 · 锦囊后端 CRUD API（数据安全级）

**现状：** `backend/server.js` 无任何 `/api/works/:workId/bible/*` 路由。IndexedDB 本地数据无法云备份，换设备全部丢失。

**需要新增的路由组（完整清单）：**

```
bible_character:
  GET    /api/works/:workId/bible/characters
  POST   /api/works/:workId/bible/characters
  PATCH  /api/bible/characters/:id
  DELETE /api/bible/characters/:id

bible_world_entry（同上结构）
bible_foreshadow（同上）

bible_glossary_term（同上）

work_style_card:

writing_style_sample（笔感样本）:
  GET    /api/works/:workId/style-samples
  POST   /api/works/:workId/style-samples
  DELETE /api/style-samples/:id

writing_prompt_template（提示词模板）:
  GET    /api/works/:workId/prompt-templates
  POST   /api/works/:workId/prompt-templates
```

**优化建议：** 考虑引入 Supabase 直连（配合 RLS）替代后端转发，可以减少约 60% 的后端样板代码，且实时同步。如果坚持用 Fastify 中转，建议提取一个通用 `makeEntityRouter(tableName, ownerField)` 工厂函数，避免八组路由重复写。

## 五、落笔页重设计建议

**当前状态：** 只是 12 个导航卡片的跳转面板，没有实际功能。

**建议方向：** 把落笔改为「写前准备台」，在真正开始写之前，这里是检查和配置创作约束的地方。

### 推荐布局

```
落笔·创作准备台
├── 顶部：当前作品选择器（切换作品后下面内容跟随变化）
│
├── 左列（约束看板）
│   ├── 风格卡（inline 编辑 POV / 调性 / 禁用词 / 风格锚点）
│   ├── 本章约束（从锦囊 chapter_bible 读取，可 inline 编辑）
│   └── 词典摘要（最近 10 条专有名词）
│
├── 中列（工具快板）
│   ├── 提示词模板（快速选择/复制到剪贴板 or 直接注入 AiPanel）
│   ├── 笔感样本（预览 + 设置「当前激活」的样本，影响所有 AI 生成）
│   └── 章模板（选择本章用哪种结构模板）
│
└── 右列（生成辅助）
    ├── 章名生成器（基于当前章节情境批量出备选）
    ├── NPC 命名器（已存在，移入此列）
    └── 伏笔清单（当前章节应该触发/埋下的伏笔清单，对照用）
```

### 各子模块具体建议

**① 风格卡（内联编辑版）**

- 当前风格卡只能在锦囊页或写作页侧栏编辑
- 落笔页直接 inline 展示可编辑的 pov / tone / bannedPhrases / styleAnchor
- 编辑时实时调用 `upsertWorkStyleCard()` 保存
- 加「测试风格约束」按钮：用当前风格卡生成一小段示例文字，确认效果

**② 提示词快板**

- 横向滚动卡片列表，显示所有 `writing_prompt_template`
- 每张卡悬浮显示完整内容，点击「复制」或「注入写作页 AI 额外要求」
- 加「新建提示词」快捷入口（inline 表单，不需要跳走）

**③ 笔感样本激活器**

- 列出所有样本，支持「设为当前激活」（写 `liubai:activeStyleSampleId` 到 localStorage）
- 激活的样本会在所有 AI 生成时自动注入上下文
- 加「从藏经书摘抽取笔感」按钮（从 ReferenceExcerpt 快速新建样本）

**④ 伏笔作战清单**

- 从 `bible_foreshadow` 读取全部伏笔条目
- 显示「未回收」的伏笔，标记哪些应在当前章节前后回收
- 支持 inline 更新伏笔状态（已埋/已收/暂挂）

**⑤ 章结构模板**

- 从 `writing_prompt_template` 中筛选 `type="chapter_structure"` 的条目
- 选中后一键填入 AiPanel 的「本章结构指令」字段

## 六、写作·正文编辑页 重设计建议

**信号：** 「不太想要」当前的写作页——值得认真对待。当前写作页的问题：

**结构症结：**

- 三栏布局（章节树 + 编辑区 + AI 侧栏）在宽屏勉强够用，窄屏极拥挤
- AI 侧栏是「手动打开的工具」，不是「写作过程中自然存在的伙伴」
- 写作体验本质上还是在用一个「带侧栏的代码编辑器」
- 零 AI 主动感知——它不知道你在写什么，除非你主动点开让它分析

**重设计方向：** 从「编辑器+工具」变为「AI 共创空间」

### 方案 A：内联 AI 副驾（推荐，改造成本中等）

保持 CodeMirror，但加入：

1. **Ghost Text 续写建议**  
   - 光标停留 1.5s 后，AI 生成灰色幽灵文字（类似 GitHub Copilot）  
   - Tab 接受，Esc 拒绝，Alt+Tab 看下一个建议  
   - 可在设置中开/关，可调触发延迟  

2. **段落级右键 AI 菜单**  
   - 选中一段文字，右键出现：扩写 / 缩写 / 改写 / 改变视角 / 检查一致性  
   - 结果显示在「浮动草稿气泡」里，用户可接受/拒绝，不打断正文  

3. **内联批注（类似 Google Docs 评论）**  
   - AI 发现潜在问题时，在行内添加波浪线 + tooltip  
   - 类型：调性漂移 / 伏笔疑似忘了收 / 人名前后不一致 / 情节逻辑疑问  

4. **沉浸写作台（改进现有 zen 模式）**  
   - 当前 zen 模式只是隐藏侧栏  
   - 加入：全屏 + 背景磨砂 + 只显示当前章 + 右下角悬浮 AI 按钮（点击展开浮层）  
   - 可选环境音（咖啡馆/雨声/白噪音）通过 Web Audio API  

### 方案 B：双栏共写界面（需要较大重构）

- 左：正文（主写作区，宽度自适应）
- 右：AI 对话流（始终可见，不需要手动打开）
  - AI 可以主动推送「章节摘要已更新」「检测到调性漂移」等通知卡片
  - 用户可以在右侧和 AI 对话，AI 产出的内容自动进入「待合并区」
  - 底部固定「待合并草稿」预览，一键合并到正文

### 方案 C：段落块编辑器（大重构，最接近 Notion）

把 CodeMirror 全文编辑器替换为块编辑器（tiptap 或 BlockNote）：

- 每个段落是独立的块
- 每个块都可以「生成/修改」这个块的内容
- 块支持「AI 写/用户写/AI 写用户改」三种状态标记
- 更适合章节级操作，但导入导出需要额外处理

**推荐：** 先做方案 A（内联 AI 副驾），保持 CodeMirror 稳定基础，用 Ghost Text + 段落右键菜单解决最大体验痛点，风险最低、效果最直接。方案 B/C 可以在产品验证后再做。

## 八、一句话总结建议

| 模块 | 核心判断 |
|------|----------|
| 推演 | 路由修了就活，功能已经很完整 |
| 锦囊 | 后端 API 是最大安全漏洞，数据随时会因为换设备丢失 |
| 写作页 | 方向不是推倒重来，是加 Ghost Text + 段落右键菜单，成本最小效果最大 |
| 生辉 | 产品差异化所在，值得专项 Sprint 攻坚，补完就是杀手级功能 |
| 落笔 | 从「导航牌」改成「写前准备台」，是整个平台的写作仪式感入口 |
| 问策 | 技法卡 + 写回链路补完后，从「AI 聊天」变成「策略智囊」 |
| 流光 | 三方式转入写作接通后，从「笔记本」变成「创作燃料箱」 |
