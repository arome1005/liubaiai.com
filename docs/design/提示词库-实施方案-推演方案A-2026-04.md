# 提示词库（独立页面）+ 推演方案 A（轻量步骤条）实施方案

> 目标：在**不大改现有推演页**（`/logic`，`V0TuiyanPage`）和写作页的前提下，引入一个可复用、可审核、可同步的「提示词资产库」，并把提示词按阶段/场景接入推演、写作等模块。
>
> 约束：**不能影响现有其他功能**。新增能力优先以「新增页面/新增数据表/新增可选入口」方式落地；旧入口保持可用。
>
> 日期：2026-04

---

## 1. 当前现状盘点（已完成 / 未完成）

### 1.1 已完成（仓库已有能力）

- **按作品维度的提示词片段（写作侧栏“额外要求”模板）**：`WritingPromptTemplate`
  - **数据模型**：`src/db/types.ts` 的 `WritingPromptTemplate`
    - 字段：`workId`、`category`（自由分类字符串）、`title`、`body`、`sortOrder`、时间戳
  - **存储**：IndexedDB（`src/storage/writing-store-indexeddb.ts`）
    - `list/add/update/delete/reorderWritingPromptTemplates`
  - **UI**：锦囊页 `BiblePage` 的 `prompts` Tab（`/work/:id/bible?tab=prompts`）
    - 可新增/编辑/排序/删除
    - 支持「去写作装配」（把模板 body 写入写作页 AI 侧栏“额外要求”）
  - **备份**：导出/导入链路包含 `writingPromptTemplates`（`src/storage/backup.ts`、`backup-merge-remap.ts`、`writing-store-indexeddb.ts` 导入逻辑）
- **推演页（方案 A 的承载页）基础骨架已存在**：`/logic`（`src/pages/V0TuiyanPage.tsx`）
  - 左：卷/章/场景树
  - 中：大纲/导图/文策三视图
  - 右：详情 / AI 对话 / 参考
  - 已有“文策”数据结构与持久化（`tuiyan_state`）

### 1.2 未完成（本次要新增/补齐）

> 下面这些是「提示词独立页面 + 审核同步 + 全模块接入」所必需的缺口。

- **独立提示词页面**（`/prompts`）与其对应的数据层（全局/跨作品）
  - 当前只有 per-work 的 `WritingPromptTemplate`，没有全局 Prompt Library 页面
- **提示词的“类型体系”**（大纲/卷纲/细纲/续写/拆书/书名等）
  - 当前 `WritingPromptTemplate.category` 是自由文本，缺少标准化类型与筛选体系
- **审核/发布/同步机制**
  - 当前没有 `draft/submitted/approved/rejected` 状态机
  - 没有“开发者审核后同步到各处”的发布层
- **跨模块接入（推演/写作/落笔/问策等）统一入口**
  - 当前只有「锦囊 prompts → 去写作装配」这一条入口
  - 推演页、落笔页等尚未使用提示词资产库（或仅硬编码 prompt）
- **变量/上下文装配（模板占位符）**
  - 当前模板仅是纯文本 body，没有变量系统；跨页面复用会受限

## 2. 目标功能清单（新增什么）

### 2.1 新增：提示词库独立页面（Prompt Library）

**页面**：`/prompts`

**核心能力**：

- **分类/类型**（可扩展）：
  - 续写、拆书、大纲、卷纲、细纲、书名、书籍简介、写作风格、黄金开篇、金手指、人设、脑洞、世界观、书籍分析……
- **搜索**：标题/标签/正文全文检索（本地先用 `includes`，后续可上索引）
- **筛选**：
  - 类型（多选）
  - 标签（多选）
  - 来源：我的草稿 / 已提交 / 已通过（发布）/ 官方
  - 排序：最新 / 收藏 / 热度（热度可先不做）
- **卡片列表**：预览、复制、收藏、使用（进入接入点）
- **新建/编辑**：支持富文本/纯文本（MVP 纯文本即可）

### 2.2 新增：提示词审核与发布（开发者审核）

**状态机**：

- `draft`（草稿，用户可立即自用）
- `submitted`（用户提交审核）
- `approved`（开发者通过，进入发布层）
- `rejected`（驳回，保留原因）
- `archived`（下架，可选）

**审核入口（MVP）**：

- 第一版可不做后台页面，先在 Supabase 控制台/脚本完成审核（前端只展示状态与原因）
- 后续再补「审核台」页面

### 2.3 新增：提示词“适用范围 / 槽位”（用于推送到各模块）

**适用范围**（多选）：

- 写作页、推演页、落笔页、问策页、生辉页、藏经页……

**槽位（Prompt Slot）**：用于“哪里会用到提示词”

- 推演：大纲生成 / 卷纲扩展 / 细纲拆场 / 文策总结（可选）
- 写作：续写 / 改写 / 扩写 / 黄金开篇 / 章节总结 / 书名简介（可选）
- 落笔：抽卡→构思母本 / 构思→大纲（可选）
- 拆书：结构分析 / 技法提炼（可选）

> 设计原则：槽位决定“入口位置”和“上下文变量”，模板本身不绑定某个页面实现细节。

### 2.4 新增：变量系统（可选但强烈建议）

**模板正文支持占位符**（例）：

- `{{work_title}}`、`{{tags}}`
- `{{chapter_title}}`、`{{chapter_content}}`
- `{{outline_node_title}}`、`{{outline_node_summary}}`
- `{{master_brief}}`（落笔构思母本）

**装配策略（MVP）**：

- 先支持少量变量（只做替换，不做复杂逻辑）
- 对缺失变量给出空字符串或提示

## 3. 与“不能大改推演”约束下的接入方案（方案 A）

### 3.1 推演页新增：轻量步骤条（不改三栏骨架）

在 `V0TuiyanPage` 顶部或右栏顶部新增一条“流程条”（可折叠）：

1. 作品构思
2. 大纲（卷→章）
3. 细纲（章→场景）
4. 完成 → 推送写作

**每一步仅做两件事**：

- 显示“本步该做什么”（文案）
- 提供“选择提示词模板”（链接到 Prompt Library 的选择器）

> 不改变现有按钮/面板的存在，仅新增“推荐路径”和“模板选择”。

### 3.2 推演页提示词接入点（最小侵入）

将推演页中与“生成”相关的入口统一支持「引用模板」：

- 左下：`AI 生成大纲`
- 章/场景相关：`生成细纲`
- 顶栏：`AI 生成`（若仍保留）

具体做法：

- 若用户已在步骤条为当前步骤选了模板，则生成时把模板正文作为前缀（或包裹）注入现有 prompt
- 若未选模板，则保持当前默认行为

### 3.3 完成后“推送到写作页”

推送策略（不破坏现有写作体验）：

- 默认：推送为 **写作页 AI 侧栏草稿/额外要求**（复用已有「写入侧栏草稿」能力）
- 可选：将细纲写入 `Chapter.outlineDraft`（仓库已有字段，`src/db/types.ts`）并在写作页提示“已导入推演细纲”

> 需要明确：推送是“可撤销/可覆盖”还是“一次性锁定”。建议 MVP 先做可覆盖但保留历史（按时间戳）。

## 4. 数据与同步（推荐 Supabase 作为发布层）

### 4.1 新表建议（Supabase）

> 命名示例：`prompt_template`（全局库），与现有 per-work `writing_prompt_template` 并存。

字段建议：

- `id`（uuid）
- `owner_user_id`
- `title`
- `type`（枚举：续写/大纲/卷纲/细纲…）
- `tags`（text[]）
- `body`（text）
- `scope`（text[]：writer/tuiyan/luobi/wence…）
- `slots`（text[]：tuiyan_outline/tuiyan_volume/tuiyan_scene/writer_continue…）
- `status`（draft/submitted/approved/rejected/archived）
- `review_note`（text，可空）
- `version`（int，更新提交用）
- `created_at/updated_at`

RLS 建议：

- `draft/submitted/rejected`：仅 owner 可读写
- `approved`：所有登录用户可读；仅 owner 可发起新提交；仅管理员可改状态

### 4.2 与现有 `WritingPromptTemplate` 的关系

- `WritingPromptTemplate`（现有）：
  - 每部作品自己的“额外要求片段”
  - 继续保留，不破坏旧入口（锦囊页 prompts）
- 新 `prompt_template`（全局）：
  - 用于跨作品、跨页面复用与审核发布
  - 可以提供“一键复制到本作品 prompts”的迁移按钮（后续）

## 5. 具体开发步骤（分迭代，不影响现有功能）

### Sprint 0（准备：不改功能，只加文档与枚举）

- 定义：
  - Prompt `type` 枚举（你列的分类）
  - Prompt `slot` 枚举（推演/写作等入口位）
  - Prompt `scope` 枚举（可用页面）
- 产出：`docs/design/提示词库-实施方案-推演方案A-2026-04.md`（本文件）

### Sprint 1（MVP：提示词库页面 + 私有模板）

目标：用户能在 `/prompts` 新建、分类、搜索、收藏、复制、使用（仅本地或 Supabase draft）。

- 新增路由与页面：
  - `src/pages/PromptsPage.tsx`（新建）
  - `src/App.tsx` 加路由
  - `AppShell` 加入口（或先从设置进入）
- 数据层：
  - 若先本地：IndexedDB 新表（不推荐长期，但可用于快速验证）
  - 推荐直接 Supabase：新增表 + RLS（并在前端拉取）
- 功能：
  - 新建/编辑/删除（仅草稿）
  - 类型/标签筛选
  - 搜索
  - 收藏（本地即可：`liubai:promptFavorites`）

验收：

- 不影响现有锦囊 prompts 与写作页“额外要求”装配

### Sprint 2（提交审核 + 发布层同步）

目标：用户提交后变为 `submitted`，你审核后 `approved`，全端可见。

- 前端：
  - 模板卡片增加“提交审核/撤回提交”
  - 展示审核状态与驳回原因
- 后端（Supabase）：
  - 管理员更新 status（MVP 可手工）
  - 后续可补管理员页面

验收：

- approved 模板在其他账号可见
- draft 模板仅自己可见

### Sprint 3（推演方案 A：步骤条 + 模板选择器接入）

目标：推演页不改骨架，只新增步骤条与模板入口，并能在生成时注入模板。

- 推演页新增组件：
  - `TuiyanFlowBar`（步骤条，可折叠）
  - `PromptPicker`（选择器：筛选 type/slot=推演相关）
- 接入点：
  - `AI 生成大纲` / `生成细纲` / 顶栏 `AI 生成`：在调用前读取当前步骤选定模板，拼接到 prompt
- 持久化：
  - 选定模板 id 存入 `tuiyan_state`（不影响现有字段）

验收：

- 不选模板时行为与现在一致
- 选模板时生成内容明显受模板影响

### Sprint 4（写作页接入 1 个样板槽位）

目标：证明“提示词能推送到写作页”。

- 写作页（AiPanel 或右侧某入口）增加：
  - 选择提示词模板（type=续写/黄金开篇/写作风格等）
  - 一键把模板 body 注入“额外要求”或作为一次请求前缀

验收：

- 不影响现有写作 AI 流程
- 用户能在写作页直接复用提示词库资产

### Sprint 5（变量系统：小步迭代）

目标：同一模板跨页面复用更强。

- 定义变量白名单与装配函数 `renderPromptTemplate(body, ctx)`
- 先支持少量变量（work/chapter/outline），逐步扩展

验收：

- 缺变量不会报错（回退为空字符串或提示）

## 6. 风险与保护措施（保证“不影响现有功能”）

- **旧提示词体系保留**：`WritingPromptTemplate`（锦囊 prompts）不删不改；新库是并行资产。
- **推演页默认不变**：步骤条可折叠；不选择模板时生成逻辑完全保持。
- **权限与安全**：发布层用 `approved` 才全端可见，避免“未审核模板污染全局”。
- **渐进接入**：先推演接 1-2 个入口，再扩展到写作/落笔/问策，避免一次改动面过大。
