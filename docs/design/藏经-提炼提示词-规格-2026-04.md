# 藏经 → 提炼提示词（Prompt）功能规格（可交付开发）

> 目标：在「藏经」本地书库（`ReferenceLibraryPage`）中，基于**整书分块**或**用户摘录**提炼出可复用的提示词模板（Prompt Template），并进入「提示词库」的**草稿**；用户可编辑并提交审核；开发者审核通过后，全体注册用户可在落笔/推演/写作等需要提示词的入口使用。
>
> 约束：不破坏现有藏经导入、分块、检索、摘录、提炼要点（characters/worldbuilding/plot_beats/craft）等功能；新增能力以“新增按钮 + 新弹窗 + 新数据类型”方式接入。
>
> 关联文档：`docs/design/提示词库-实施方案-推演方案A-2026-04.md`
>
> 日期：2026-04

---

## 1. 名词与范围

### 1.1 名词

- **藏经**：本地参考书库（`ReferenceLibraryPage`），书籍文本按 chunk 存储，支持摘录与搜索。
- **摘录（Excerpt）**：用户在阅读器中圈选产生的片段，存在 `ReferenceExcerpt`。
- **提炼要点（Extract）**：现有功能，整书 chunkTexts → 生成 characters/worldbuilding/plot_beats/craft 文本，保存为 `ReferenceExtract`。
- **提示词模板（Prompt Template）**：新增资产。类型化（续写/大纲/细纲等），可带变量占位符，具有状态机（draft/submitted/approved/rejected）。
- **槽位（Prompt Slot）**：提示词被使用的位置（落笔/推演/写作的某个功能入口）。

### 1.2 本规格覆盖范围（MVP + V1）

**MVP（优先交付）**：

- 从**摘录**提炼为提示词（最稳、成本低）
- 从**整书 chunkTexts**提炼为提示词（复用现有提炼调用管线）
- 生成后落到提示词库为 `draft`，并提示“去提示词库完善/提交审核”

**V1（可选增强）**：

- 多摘录合并提炼（1-5 条摘录）
- 预设“风格要素”提炼（节奏/句式/镜头感等结构化输出）
- 提炼进度条（整书提炼耗时较长）

**明确不做（本期）**：

- 自动“复刻某作者/某 IP 原文风格”的直接宣称与模板命名（合规/风控）
- 提炼结果自动写入正文（只进入提示词库草稿）
- 后台审核台页面（MVP 可由开发者手工审核；前端仅展示状态）

## 2. 用户故事（User Stories）

### US-01：从摘录提炼“续写提示词”

1. 用户在藏经阅读器中选中一段打斗片段并保存为摘录（已存在能力）。
2. 在摘录列表中点击「提炼为提示词」。
3. 选择类型：`续写`，选择目标槽位：`写作-续写`，填写模板名称（可默认）。
4. 点击「开始提炼」。
5. 系统生成一个提示词模板草稿（draft），展示预览，并给出按钮「保存到提示词库」。
6. 保存后提示“已生成草稿，去提示词库完善并提交审核”。

### US-02：从整书提炼“大纲/细纲提示词”

1. 用户在藏经书目卡片或阅读器中点击「提炼提示词」。
2. 选择类型：`大纲` 或 `细纲`，选择目标槽位：`推演-大纲生成` 或 `推演-细纲拆场`。
3. 点击开始，系统读取该书 chunkTexts 并提炼，生成草稿模板。

### US-03：审核发布后全员可用

1. 用户在提示词库中提交审核。
2. 开发者审核通过后，该模板状态变为 `approved`。
3. 其他注册用户在落笔/推演/写作的提示词选择器里可搜索并使用。

## 3. UI 设计与交互（藏经页侧）

### 3.1 入口位置（MVP）

**入口 A：摘录列表**（强推荐）

- 在每条摘录卡片的操作区新增按钮：`提炼为提示词`
- 位置：与“跳转阅读器/编辑/删除”等按钮同级（不改变既有布局，仅新增一颗按钮）

**入口 B：提炼要点面板**（可选）

- 现有“提炼要点”面板（extractPanel）顶部增加按钮：`提炼提示词（Beta）`
- 作用：整书 chunkTexts 的提炼入口

### 3.2 弹窗：提炼为提示词（Prompt Extract Dialog）

**标题**：提炼为提示词

**步骤**：

1. **来源（只读展示）**
  - 来源书名：`activeTitle`
  - 来源范围：
    - 摘录：显示摘录摘要（前 80 字）+ 备注
    - 整书：显示“整本书（分块 N 段）”
2. **类型（必选）**
  - 单选列表（MVP 先给 8 个）：
    - 续写（continue）
    - 写作风格（style）
    - 黄金开篇（opening）
    - 打斗/动作（action_scene）*（可映射到“续写”但更专用）
    - 大纲（outline）
    - 卷纲（volume_outline）
    - 细纲（scene_outline）
    - 拆书/技法（analysis）
  - V1 扩展：书名、书籍简介、世界观、人设、金手指、脑洞等
3. **适用范围（多选）**
  - 复选：落笔 / 推演 / 写作 / 问策 / 生辉（MVP 至少支持：落笔、推演、写作）
4. **槽位（多选，受适用范围联动过滤）**
  - 写作：
    - writer_continue（续写）
    - writer_rewrite（改写）
    - writer_opening（黄金开篇）
  - 推演：
    - tuiyan_outline（大纲生成）
    - tuiyan_volume（卷纲扩展）
    - tuiyan_scene（细纲拆场）
  - 落笔：
    - luobi_master_brief（抽卡→构思母本）
    - luobi_to_outline（构思→大纲）
5. **模板名称（必填）**
  - 默认值策略：
    - `【藏经·<书名>】<类型中文名> · <一句风格标签>`
  - 可手动改
6. **标签（可选）**
  - 默认：`["藏经", "<书名>"]`
7. **开始提炼**
  - 按钮：`开始提炼`
  - loading：显示流式输出区（复用现有 extractStreaming UI 风格）
  - 取消：`停止`（AbortController）
8. **结果预览**
  - Tab：`模板正文` / `风格要点`（若有）/ `变量与说明`
  - 按钮：`保存到提示词库（草稿）`
  - 次按钮：`复制`

### 3.3 空态/错误态

- 无摘录：按钮不显示或禁用并提示“请先添加摘录”
- AI 未配置/Key 缺失/隐私守卫：沿用现有 AI 错误提示规范（不吞错误）
- 结果为空：提示“提炼结果为空，请换一段摘录或调整类型”

## 4. 提炼输出格式规范（强制模板化，便于复用）

> 核心目标：提炼结果不是“随便一段话”，而是可直接拿来当 Prompt Template 的可控格式。

### 4.1 通用输出结构（必须）

提炼结果应输出以下 3 段（Markdown）：

1. `## 任务`
2. `## 输入（变量）`
3. `## 输出要求`

示例骨架：

```md
## 任务
<描述要生成什么>

## 输入（变量）
- 书名/参考：{{ref_title}}
- 写作目标：{{work_title}} / {{chapter_title}}
- 正文（可选）：{{chapter_content}}
- 额外要求（可选）：{{user_hint}}

## 输出要求
- <结构化要求 1>
- <结构化要求 2>
...
```

### 4.2 各类型的“输出要求”最低规范

#### A) 续写（writer_continue）

- 输出为**可直接接续正文**的文本
- 必须包含：
  - 节奏要求（短句/长句比例建议）
  - 打斗/动作（若适用）镜头调度要点
  - 禁忌（不要复述前文、不要流水账等）

#### B) 写作风格（style）

- 输出为一段“约束块”，可注入系统提示或额外要求
- 必须包含：
  - 句式偏好、用词倾向、比喻密度、情绪张力曲线
  - 反例（不要写成什么）

#### C) 大纲（tuiyan_outline）

- 输出必须是结构化列表（Markdown）
- 最低规范：
  - 3-6 卷
  - 每卷 6-15 章
  - 每章一句“剧情推进 + 冲突点/信息增量”

#### D) 卷纲（tuiyan_volume）

- 输入：当前卷主题 + 目标
- 输出：
  - 本卷 6-12 章标题与一句话
  - 本卷三段节拍：开端/发展/高潮（每段 2-3 条要点）

#### E) 细纲（tuiyan_scene）

- 输入：章标题 + 章目标 + 已有摘要
- 输出：
  - 6-12 个场景
  - 每场景：目的、冲突、信息增量、结尾钩子、建议字数

#### F) 拆书/技法（analysis）

- 输出：
  - 可复用技法清单（5-10 条）
  - 每条包含：何时用、如何用、常见失败点

## 5. 与提示词库（/prompts）与审核发布的衔接

### 5.1 生成后的落库规则

- 生成后默认保存为：
  - `status = draft`
  - `source = reference_extract`（新增字段或 tags 里带 `source:ref`）
  - `refWorkId`（可选：记录来源书 id，便于追溯）

### 5.2 提交审核

- 提示词库中提供：`提交审核`
- 状态展示：
  - draft：可编辑、可删除
  - submitted：只读（或允许编辑但会产生新版本），显示“等待审核”
  - rejected：显示原因，可修改后再次提交
  - approved：全员可见

---

## 6. 数据结构（建议）

> 若提示词库已按全局表 `prompt_template` 落地，建议额外补充字段用于追溯来源。

`prompt_template` 建议新增/约定字段：

- `source_kind`: `"manual" | "reference_excerpt" | "reference_book"`
- `source_ref_work_id`: string | null
- `source_excerpt_ids`: string[] | null（V1 多摘录时启用）
- `source_note`: string | null（例如“提炼类型/参数”）

## 7. 实现步骤建议（可直接拆任务）

### Phase 1（MVP：摘录→提示词草稿）

- `ReferenceLibraryPage` 摘录卡片新增按钮「提炼为提示词」
- 新增 `PromptExtractDialog`（组件）
- 复用现有 AI 管线（参考 `extractReferenceContent` 的 onDelta/AbortController）
- 将提炼结果保存为 prompt_template（draft）
- 提示“去提示词库查看”

### Phase 2（整书→提示词草稿）

- 在书目卡片/提炼面板增加入口「提炼提示词」
- 读取 chunkTexts（现有逻辑可复用）
- 提炼并保存

### Phase 3（接入落笔/推演/写作选择器）

- 落笔页提示词选择器：能看到 draft（本人）+ approved（全体）
- 推演页步骤条（方案 A）模板选择器过滤 slot=tuiyan_*
- 写作页（AiPanel）样板槽位接入（writer_continue 或 opening）

---

## 8. 验收标准（Acceptance Criteria）

- 在藏经页对任意摘录点击「提炼为提示词」，能生成可读的模板正文，并保存为草稿。
- 草稿在提示词库中可编辑、可提交审核。
- 审核通过后，其他账号能在落笔/推演/写作的提示词选择器中找到并使用。
- 现有藏经“提炼要点/导入锦囊/笔感样本”等功能不受影响。
