# 推演 — 大纲与逻辑规划（v0 目标态）

> 路由：`/logic`  
> UI 口径：**最终 v0 对齐后的三栏工作台**（大纲 / 导图 / 文策 + 右侧详情/AI对话/参考）。  
> 功能口径：**最终全部功能均已实现**（不写“待开发/占位”）。

---

## 1. 页面定位

- **一句话目的**：围绕作品的大纲结构、逻辑一致性与改纲决策，提供可追溯的规划与迭代工作台，并把产物无缝推进到 **写作页** / 问策 / **锦囊** / 生辉（「落笔」为 §5 总域名，见 `README` 术语表）。
- **主要用户任务**：
  - 构建/维护卷-章-场景的大纲树（结构化规划）
  - 执行一致性扫描（规则 + AI 语义）并回收为“可处理问题”
  - 生成剧情三分支并进一步：写入写作草稿 / 去问策跟进
  - 用时间线与地点/地图面板维护“可检索的事实”（供扫描与写作注入）
  - 用文策记录决策、修订、AI 建议（可置顶、可追溯）
  - 从藏经引用提炼要点并应用到大纲/文策

## 2. 入口与导航

### 2.1 进入本页

- **主入口**：Hub 顶栏“推演”。
- **作品直达入口**：作品库（留白）作品卡片的“推演”快捷入口。
- **深链进入（最终态要求）**：
  - 支持从其它页面携带“优先作品/优先章节”进入（例如 `navigate state.preferWorkId`）。
  - 若从 **写作页** 进入推演，默认选中当前作品与当前章节（或最近章节），并定位到对应的大纲节点（若已建立映射）。

### 2.2 从本页离开（去哪里）

- **去写作页**：
  - 在大纲节点“进入生辉 / 进入写作 / 应用到正文”等动作后跳转 `/work/:id` 并保留上下文（选中节点、建议草稿、引用材料）。
- **去问策**：
  - 从三分支预测/一致性告警/冲突点/AI 对话一键“去问策跟进”，打开 `/chat` 并预填内容与关联作品。
- **去锦囊**：
  - 从一致性扫描结果/引用字段跳转到 `/work/:id/bible` 的对应板块，并可高亮相关条目。
- **去藏经**：
  - 从右侧“参考”面板进入 `/reference` 的特定条目或检索页（最终态支持双向跳转）。

### 2.3 状态保留（最终态要求）

- **记忆项**：
  - 当前作品、当前章节（若存在章选择）
  - 当前视图：大纲/导图/文策
  - 左右栏显隐
  - 右侧栏 Tab：详情/AI 对话/参考
  - 大纲树折叠状态、搜索词
  - AI 扫描范围（本章/最近 5 章/全书）与“忽略项显示”开关
- **返回行为**：
  - 从推演跳去问策 / **写作页** 再返回时，以上状态保持（除非用户显式切换）。

## 3. 信息架构（v0 目标 UI）

> 推演页是典型三栏：**左：大纲树**，**中：主视图**，**右：详情/AI/参考**。顶部为统一工具栏。

### 3.1 顶部工具栏（Top Toolbar）

从左到右的固定结构（对齐 v0 `tuiyan-module.tsx`）：

- **左侧栏开关**：收起/展开大纲树（带 tooltip）。
- **作品选择器**：
  - 展示：作品名 + 标签（如题材）+ 下拉箭头
  - 下拉内容：最近作品/全部作品（最终态）+ “新建推演项目”
- **视图切换（三段）**：大纲 / 导图 / 文策
- **定稿进度**：百分比 + 进度条（按“已定稿节点数 / 总节点数”）
- **动作区**：
  - 导出大纲（tooltip）
  - 推演设置（tooltip）
  - AI 生成（主按钮）
- **右侧栏开关**：收起/展开右侧面板（带 tooltip）

显示条件：

- 无论是否选中作品，顶部工具栏始终显示。
- 当无作品时，作品选择器提供“新建作品/导入作品”的跳转入口（最终态可复用留白能力）。

### 3.2 左侧栏：大纲树（Outline Panel）

结构（从上到下）：

- **搜索框**：搜索大纲节点标题/标签
- **统计条**：卷/章/场景/已定（四项）
- **大纲树**：
  - 节点层级：卷 → 章 → 场景
  - 节点信息：类型图标（卷/章/场景）+ 标题 + 状态图标（草稿/打磨/已定稿/锁定）
  - 展开/折叠：有子节点时可折叠
  - 选中态：高亮当前节点
  - 快捷菜单（`...`）：
    - 编辑
    - 添加子项
    - 更改状态（子菜单：草稿/打磨中/已定稿/锁定）
    - AI 优化
    - 生成细纲
    - 删除（危险）
- **底部动作**：
  - 新建卷
  - AI 生成大纲

### 3.3 中间主栏：三视图（Main Panel）

#### 3.3.1 大纲视图（Outline Detail）

选中节点后展示的核心卡片区（对齐 v0）：

- 节点类型徽标（卷/章/场景）
- 节点状态徽标（草稿/打磨/已定稿/锁定）
- 标题 + `...` 更多菜单（复制/锁定/删除等）
- 内容摘要卡（支持编辑）
- 标签区（展示标签 + 添加入口）
- 情绪曲线（若存在）
- 冲突点列表（若存在）
- 登场人物 / 涉及地点（若存在）
- 目标字数卡（若存在）
- 快捷动作按钮：
  - 添加子项
  - 生成细纲
  - AI 优化
  - 进入生辉（与 **写作页** / 锦囊材料联动）

并在大纲视图的“主栏下方/侧边”嵌入推演页的真实能力（最终态整合要求）：

- **规则一致性扫描结果区**：以告警卡片列表呈现（可定位到问题段落/对应锦囊条目/术语/禁忌）
- **LLM 语义扫描结果区**：同上，但支持忽略/恢复与范围选择
- **三分支预测区**：三条分支卡片（标题 + 摘要 + 操作：写入草稿 / 去问策跟进）
- **事实面板入口**：时间线/地点/地图（在 v0 UI 的合理区块内展示，详见“功能清单”）

> 最终态要求：这些真实能力必须“看起来就是 v0 推演页的一部分”，而不是另起一套布局；但信息架构仍保持 v0 的主骨架（左树/中栏/右栏）。

#### 3.3.2 导图视图（Mindmap）

- 中心节点：作品
- 两侧节点：卷节点（左右布局）
- 连线：弱可视化（装饰性）
- 底部导图工具条（fixed）：
  - 添加节点
  - 视图放大（Maximize）
  - 刷新/重新布局
  - 导出图片

#### 3.3.3 文策视图（WenCe Log）

- 页头：标题“文策日志”+ 说明 + 新建记录按钮
- 过滤条：全部/决策/修订/AI建议/灵感/里程碑
- 置顶区：显示所有置顶记录（带 Pin 标识）
- 时间线区：按时间倒序的文策卡片列表
- 文策卡片结构（对齐 v0）：
  - 类型图标（决策/修订/AI建议/灵感/里程碑）
  - 时间戳
  - 标题 + 内容
  - tags（以 `#tag` 样式展示）
  - `...` 菜单：置顶/取消置顶、复制、删除

### 3.4 右侧栏：详情 / AI 对话 / 参考（Right Panel）

右侧栏顶部为 Tab（对齐 v0：三等分 + 激活下划线）：

- **详情**：
  - 快速编辑（标题/摘要）
  - 子项预览列表（点击跳转选中）
  - 操作区（四宫格按钮）：
    - AI 扩写
    - 生成细纲
    - 标记定稿
    - 进入生辉
- **AI 对话**：
  - 对话消息区（滚动）
  - 快捷提示词按钮：优化节奏/完善人物/强化冲突/理清逻辑
  - 输入框（支持 Enter 发送、Shift+Enter 换行）
  - 状态：生成中气泡（动效三点）
  - 已选上下文标签：显示当前选中节点
  - 发送按钮（禁用态与 loading）
- **参考**：
  - 关联藏经条目列表（书名/作者/已提炼）
  - 每本书的提炼要点列表（人物/世界观/情节/技法）
  - “应用到当前大纲”（把要点转换为建议或直接写入文策/节点字段）
  - 底部引导：浏览藏经（跳转 `/reference`）

## 4. 功能清单（角落级）

> 本节按最终态把“v0 UI 组件”与“现有推演真实能力”逐条写清，并明确联动来源/写回位置。

### 4.1 作品与章节选择

- **作品选择器（顶部）**：
  - 切换作品后：左侧大纲树、主栏内容、右侧面板全部刷新到该作品数据域。
  - 默认选中章节/节点策略：
    - 若从 **写作页** 进入：优先当前章节对应的大纲节点
    - 否则：优先最近编辑章节/最近选中节点
- **章节选择（最终态）**：
  - 作为“扫描/三分支/时间线快速追加”的上下文来源。
  - 入口位置：可放在主栏的“扫描/生成”区块中（按 v0 视觉落点）。

### 4.2 大纲树节点管理（卷/章/场景）

- **新建卷**：
  - 入口：左侧栏底部
  - 行为：创建卷节点并进入编辑标题；默认状态草稿
  - 写回：推演大纲数据结构（作品域）
- **添加子项**：
  - 入口：节点 `...` 菜单或主栏“添加子项”
  - 行为：在当前节点下新增子节点（卷下加章、章下加场景）
  - 写回：推演大纲
- **编辑节点**：
  - 入口：节点 `...` 菜单“编辑”或主栏“编辑”
  - 可编辑字段：标题、摘要、标签、冲突点、情绪曲线、人物/地点关联、目标字数
  - 写回：推演大纲（并触发进度/统计更新）
- **更改状态**：
  - 入口：节点 `...` 菜单 → “更改状态”子菜单
  - 状态定义：
    - 草稿：结构未稳定
    - 打磨中：内容迭代中
    - 已定稿：可作为写作依据，计入定稿进度
    - 锁定：禁止编辑（需要解锁动作）
  - 联动：定稿进度更新；右侧“标记定稿”按钮与此一致
- **锁定/解锁**：
  - 入口：主栏 `...` 菜单（最终态）
  - 行为：锁定后，编辑入口变为 disabled 并提示原因
- **删除节点**：
  - 入口：节点 `...` 菜单“删除”
  - 规则：删除带子项节点需二次确认（并明确级联删除）
  - 写回：推演大纲；若删除的是与章节映射的节点，需同步解除映射

### 4.3 规则一致性扫描（规则扫描）

> 来自现有功能清单：**比对正文 vs 锦囊禁写、风格卡禁用套话、术语表、人物禁忌**。

- **入口**：主栏“扫描”按钮（最终态可在右侧详情或主栏顶部动作区出现）。
- **前置条件**：需选中作品与目标章节（或范围）。
- **扫描范围**：
  - 默认：当前章节
  - 可扩展：最近 N 章 / 全书（与 AI 语义扫描保持一致的选择器）
- **输出（UI）**：告警卡片列表，每条告警包含：
  - 告警类型（禁写/禁忌/术语/风格套话等）
  - 命中片段摘要
  - 严重级别
  - 操作：
    - 定位到 **写作** 正文（打开 `/work/:id` 并滚动 / 高亮）
    - 打开锦囊对应条目（`/work/:id/bible` 并定位）
    - 一键修复建议（写入 AI 对话或写入草稿区）
- **写回**：扫描结果以“推演结果/问题列表”的形式存储，支持跨会话可追溯与在文策中留痕（最终态）。

### 4.4 LLM 语义扫描（AI 扫描）

> 来自现有功能清单：调用 AI 分析逻辑矛盾；可选范围；可取消；结果可忽略/恢复。

- **入口**：主栏“AI 语义扫描”按钮（最终态在推演工具区块）。
- **范围选择**：本章 / 最近 5 章 / 全书
- **运行过程**：
  - 展示生成中状态，可取消（Abort）
  - 显示“本次使用材料”摘要（最终态与 AiPanel 口径一致）
  - 成本/高危确认：当预估 token 超阈值需二次确认（与全局成本策略一致）
- **输出（UI）**：语义矛盾列表（每条包含：矛盾描述、证据片段、建议改法、可信度）
- **忽略/恢复**：
  - 每条可忽略并可切换“显示已忽略”
  - 忽略行为写入可追溯记录（文策或扫描日志）
- **联动**：
  - 一键“转问策”：把矛盾作为对话议题
  - 一键“写入草稿”：把建议写入写作侧栏草稿（在 **写作页** 合并入正文）

### 4.5 三分支预测（剧情分支）

> 来自现有功能清单：输入倾向提示 → AI 生成三条走向（标题+摘要）；可写入草稿/去问策跟进。

- **入口**：主栏“三分支预测”区块（输入倾向提示 + 生成按钮）。
- **生成结果**：三张分支卡（A/B/C）：
  - 标题 + 摘要
  - 操作按钮：
    - **写入写作侧栏草稿**：写入 `liubai:aiPanelDraft`（以实现口径）
    - **去问策跟进**：打开 `/chat` 并预填分支内容（含关联作品/章节/节点）
    - **加入文策**：把该分支作为“决策候选”写入文策（最终态）
- **联动来源**：风格卡 + 锦囊索引 + 章节上下文 + 标签侧写（最终态注入规则在数据总览中统一说明）。

### 4.6 时间线（锦囊时间线快速维护）

> 来自现有功能清单：展示锦囊时间线事件列表；可快速追加事件（标签/备注/可选关联章）。

- **入口**：推演主栏的“事实面板”区块（或右侧详情/参考的子区块，最终按 v0 落点）。
- **展示**：时间线事件列表（按时间排序，可按标签筛选）。
- **快速追加**：
  - 字段：事件名称、备注、标签、可选关联章节
  - 写回：`BibleTimelineEvent`
- **联动**：
  - 规则扫描与 AI 扫描可引用时间线事件作为“事实基线”
  - **写作页** AiPanel 注入可选包含时间线摘要

### 4.7 地图与地点事件（Place & Map）

> 来自现有功能清单：SVG 地图 + 地点列表 + 事件列表；新增地点（名称/备注/坐标），新增事件（可关联章节）。

- **入口**：推演主栏“事实面板”中的“地点/地图”子视图（最终态）。
- **地点列表（LogicPlaceNode）**：
  - 新增地点：名称、备注、坐标（或从地图点选）
  - 编辑地点：名称/备注/坐标
  - 删除地点：二次确认（若有关联事件需说明级联）
- **事件列表（LogicPlaceEvent）**：
  - 新增事件：名称、备注、关联地点、可关联章节
  - 编辑/删除事件
- **地图视图**：
  - SVG 地图展示地点坐标
  - 拖拽/点选调整坐标（最终态）
- **联动**：
  - 扫描结果可引用地点事件作为事实
  - **写作页**可从地点事件一键插入描写建议（走草稿区）

### 4.8 右侧 AI 对话（推演内对话）

- **入口**：右侧栏 Tab“AI 对话”。
- **上下文来源（最终态要求）**：
  - 当前作品 + 当前章节 + 当前选中大纲节点
  - 选中的锦囊索引（人物/世界观/术语/禁忌）
  - 最近扫描结果摘要（可选）
  - 藏经引用要点（可选）
- **快捷提示词**：优化节奏/完善人物/强化冲突/理清逻辑
- **输出**：
  - 生成建议
  - 可选：结构化建议（可采纳）→ 直接写回大纲字段或写入文策（最终态）

### 4.9 导出与设置

- **导出大纲**（顶部下载按钮）：
  - 导出内容：大纲树（含节点字段）、文策日志、扫描结果摘要（最终态）
  - 导出格式：JSON/Markdown/图片（导图）等（按 v0）
- **推演设置**（顶部齿轮按钮）：
  - 扫描与 AI 默认范围
  - 成本阈值与确认策略（只展示与推演相关的）
  - 文策记录策略（自动记录哪些动作）

## 5. 数据与写回（读/写/缓存）

### 5.1 读取（与现有产品说明书对齐）

- `Work`、`Chapter`
- `ChapterBible`（本章锦囊字段）
- `WorkStyleCard`（风格卡）
- `BibleCharacter`、`BibleWorldEntry`、`BibleTimelineEvent`、`BibleGlossaryTerm`
- `LogicPlaceNode`、`LogicPlaceEvent`
- 推演产物：大纲节点树、文策日志、扫描结果、三分支结果（最终态可持久化）

### 5.2 写回

- 大纲节点树（卷/章/场景）及其字段
- 文策日志（含置顶、删除、复制）
- 扫描结果（规则/AI），以及忽略/恢复状态
- 三分支结果（持久化与引用）
- 时间线事件（快速追加写入 `BibleTimelineEvent`）
- 地点与事件（写入 `LogicPlaceNode` / `LogicPlaceEvent`）
- 写作侧栏草稿（跨模块）：`liubai:aiPanelDraft...`

### 5.3 缓存与一致性（最终态要求）

- 任何 AI 结果必须可追溯（至少能在文策/扫描日志中看到“何时生成、基于什么材料”）。
- 写回失败必须可重试且不丢输入（尤其是长文本字段与 AI 结果采纳）。

## 6. 跨模块联动（因果链）

### 6.1 推演 → 写作页

- **触发点**：
  - 三分支“写入草稿”
  - 扫描结果“一键修复/写入草稿”
  - 右侧“进入生辉/进入写作”
- **落点表现**：
  - 打开 `/work/:id`，AiPanel 草稿区出现对应草稿
  - 或定位到正文片段并高亮（针对扫描命中）
- **来源**：推演产物 + 锦囊索引 + 风格卡注入

### 6.2 推演 → 问策

- **触发点**：三分支“去问策跟进”/扫描结果“转问策”/AI 对话“一键形成议题”
- **落点表现**：打开 `/chat`，新会话预填内容（含关联作品/章节/节点引用）
- **来源**：推演上下文 + 文策/扫描条目

### 6.3 推演 ↔ 锦囊

- **推演读锦囊**：规则扫描与 AI 扫描注入人物/术语/禁忌/时间线
- **推演写锦囊**：时间线快速追加写入 `BibleTimelineEvent`；以及（最终态）从推演形成的“设定条目建议”可一键写入锦囊
- **跳转**：扫描结果可跳转到锦囊对应条目并高亮

### 6.4 推演 ↔ 藏经

- **推演读藏经**：右侧“参考”引用书籍提炼要点
- **推演写回**：
  - 应用要点到当前大纲（写入节点字段/标签/冲突点）
  - 生成文策记录（引用来源书籍）

### 6.5 推演 ↔ 生辉

- **推演 → 生辉**：节点“进入生辉”带上该节点摘要/风格/人物地点，作为仿写输入上下文
- **生辉 → 推演**：生辉生成后可反向生成“节点改写建议”并写入文策（最终态）

## 7. 空态 / 加载态 / 错误态

### 7.1 空态

- **无作品**：提示先去留白新建/导入作品，并提供跳转按钮。
- **无章节**：提示先去 **写作页** 建立章节；推演的扫描 / 三分支按钮禁用并说明原因。
- **无大纲节点**：提示“先新建卷/AI 生成大纲”；主栏显示引导卡。
- **无扫描结果**：扫描区显示“暂无结果”，并提示如何运行扫描。

### 7.2 加载态

- 切换作品/章节时：左树与主栏 skeleton；右侧面板保持但显示加载占位。
- AI 运行中：按钮 loading + 可取消；消息区/结果区显示生成中占位。

### 7.3 错误态与自救

- **读写失败**：显示“重试”；必要时提示检查网络/同步设置。
- **AI 错误**：
  - 配置错误（Key/模型不可用）：给“打开设置”入口
  - 上下文过长：提供“精简并重试”（自动缩短注入）
  - 成本阈值：要求数字确认
- **写回冲突**：提示“数据已更新，是否重新载入/合并”，并保留用户输入不丢失。

## 8. 验收与手测脚本（可执行）

> 以“最终态全功能”为验收目标；每条可直接由测试执行。

### 8.1 基础路径（必过）

1. 从 `/library` 进入某作品的“推演”，确认作品选择器为该作品。
2. 左侧大纲树可折叠/选中；右侧面板可收起/展开。
3. 在顶部切换：大纲/导图/文策三视图，确认状态保持与渲染正确。
4. 在右侧切换：详情/AI 对话/参考，确认 Tab 下划线与内容正确。

### 8.2 大纲编辑（必过）

1. 新建卷 → 添加章 → 添加场景；确认树结构正确且状态默认草稿。
2. 更改状态为“已定稿”，确认顶部定稿进度变化。
3. 编辑节点摘要/标签/冲突点，确认主栏与右侧详情同步显示。

### 8.3 扫描与三分支（必过）

1. 运行规则一致性扫描，确认输出告警卡并可跳转到 **写作页** / **锦囊** 对应位置。
2. 运行 LLM 语义扫描（本章/最近 5 章/全书三种范围），确认可取消、可忽略/恢复。
3. 生成三分支预测，确认每条分支支持：
  - 写入写作侧栏草稿（进入 `/work/:id` 可看到草稿）
  - 去问策跟进（进入 `/chat` 预填内容）
  - 加入文策（文策视图出现记录）

### 8.4 时间线与地点（必过）

1. 在推演中快速追加时间线事件，进入 `/work/:id/bible` 的时间线板块确认事件写回成功。
2. 新增地点与地点事件，地图上可见并可编辑；扫描/AI 对话能引用地点事实（最终态）。

### 8.5 参考与引用（必过）

1. 在右侧“参考”选择一本藏经条目，查看提炼要点。
2. 点击“应用到当前大纲”，确认：
  - 大纲节点字段/标签/文策记录发生变化
  - 可追溯来源（书名/条目）
