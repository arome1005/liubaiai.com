# 生辉 — AI 仿写生成（v0 目标态）

> 路由：`/sheng-hui`  
> UI 口径：**最终 v0 对齐后的生辉工作台**（上下文装配 + 快捷 prompt + 多版本草稿 + 快照/采纳 + 写回写作）。  
> 功能口径：**最终全部功能均已实现**（不写“待开发/占位”）。

---

## 1. 页面定位

- **一句话目的**：把推演/锦囊/风格卡/前情摘要等材料装配成“可控的仿写输入”，生成多个草稿版本，并把采纳结果可靠写回到**写作页**的草稿区/正文合并流程。
- **主要用户任务**：
  - 选择作品/章节与生成目标（续写段落 / 重写选中 / 扩展细节 / 优化对话 / 生成全章）
  - 确认上下文材料是否加载齐全（细纲/锦囊/风格/前情/上一章片段）
  - 调整生成参数（温度、长度、风格强度等）
  - 生成多个版本并对比（Diff / 对比视图）
  - 采纳/不采纳、评分（赞/踩）并形成快照历史
  - 一键写入写作侧栏草稿，进而合并入正文

## 2. 入口与导航

### 2.1 进入本页

- **主入口**：Hub 顶栏“生辉”。
- **从推演进入**：
  - 大纲节点“进入生辉”
  - 细纲已定稿后“一键生成章节”
  - 推演结果（冲突点/情绪曲线/人物地点）作为生辉上下文项
- **从写作页进入**：
  - 顶栏“生辉/仿写”入口
  - 选区“重写/润色”跳入生辉，并携带选区文本与插入策略
- **从流光进入**：碎片“转为仿写提示”跳入生辉（最终态）

### 2.2 从本页离开（去哪里）

- **写回写作**：采纳草稿或“写入草稿” → `/work/:id` 并打开 AiPanel 草稿区/合并弹窗
- **去推演**：查看细纲/节点 → `/logic` 定位对应节点
- **去锦囊**：从上下文项跳转 `/work/:id/bible` 定位条目（修正设定后再生成）
- **去问策**：将草稿差异/卡点“一键问策” → `/chat`

### 2.3 状态保留（最终态要求）

- 记忆项：
  - 当前作品、当前章节、当前选区（若来自写作页）
  - 当前 prompt（输入框）
  - 上下文勾选项与加载状态
  - 参数（温度、长度等）
  - 草稿版本列表与当前选中版本
  - 已采纳版本标记与评分
- 返回行为：从写作页合并后返回生辉，仍保留历史快照与“当前采纳”标记。

## 3. 信息架构（v0 目标 UI）

> v0 `shenghui-module.tsx` 的骨架是：顶部工具条 + 左侧上下文/细纲节点 + 中间生成与草稿区 + 右侧版本/对比/设置面板（可收起）。

### 3.1 顶部工具条（Top Bar）

- **作品/章节上下文**：显示当前作品、章节（可切换）
- **模型选择器**（若提供）与费用提示（Coins）
- **快捷动作**：
  - 生成（Send / Sparkles）
  - 重试/刷新（RefreshCw）
  - 版本历史（History）
  - 右侧面板开关（PanelRight / PanelRightClose）
- **状态区**：生成中 Loader、错误提示 AlertCircle、成本/Token 概览

### 3.2 左侧栏：上下文与细纲（Context）

左侧强调“材料齐不齐”：

- **上下文项列表（ContextItem）**：
  - 细纲（outline，必选）
  - 锦囊（bible，必选）
  - 文策风格（style，必选）
  - 风格卡（style，必选）
  - 前情摘要（summary，可选）
  - 上一章片段（chapter，可选，可能未加载）
  - 每项显示：label、value、tokens、loaded 状态、优先级（required/optional）
  - 操作：展开查看、重新加载、禁用（可选项）
- **细纲节点列表（OutlineNode）**：
  - 每段：标题、摘要、状态（draft/finalized/generated）、目标字数/当前字数
  - 操作：选中某段作为生成目标；或一键生成整章（按节点顺序）

### 3.3 中间主栏：生成区与草稿展示（Draft）

- **系统提示条**：提示“已加载哪些材料”（可折叠）
- **快捷 prompt 按钮**：
  - 续写段落 / 重写选中 / 扩展细节 / 优化对话 / 生成全章 / 达成目标
  - 点击后填入输入框（不立即发送）
- **prompt 输入区**：
  - 多行输入
  - Enter 发送、Shift+Enter 换行
- **生成结果区**：
  - 草稿正文（可滚动）
  - 可视化标记：章节段落结构、引用提示（最终态）

### 3.4 右侧栏：版本/对比/设置（Right Panel）

- **版本列表（GeneratedDraft）**：
  - version、时间、tokens、cost、temperature
  - 评分（赞/踩）
  - “当前采纳”标记（isAccepted）
  - 删除版本（Trash2）
- **对比（Diff / GitCompare）**：
  - 版本 A vs 版本 B
  - 或 “草稿 vs 当前正文”（需要从写作页拉取当前章文本）
- **写回动作**：
  - 写入写作草稿
  - 直接合并到正文（走写作合并弹窗）
  - 存为流光碎片（最终态）
- **参数设置**：
  - 温度（Slider）
  - 长度/目标字数（Progress/输入）
  - 是否严格遵循细纲（Switch）
  - 是否注入设定索引（Switch，等价 includeSettingIndex）

## 4. 功能清单（角落级）

### 4.1 作品/章节选择与生成目标

- **作品切换**：
  - 入口：顶部作品选择器
  - 结果：上下文项重新装配（细纲/锦囊/风格卡/摘要）；草稿版本按作品域隔离
- **章节切换**：
  - 入口：章节选择器
  - 结果：细纲节点、前情摘要、上一章片段刷新
- **选区生成（来自写作页）**：
  - 入口：写作选区“重写/润色”跳入生辉
  - 行为：默认 quick prompt 为“重写选中”，并将选区文本作为 required 上下文项

### 4.2 上下文项管理（Context Builder）

- **必选项（required）**：细纲、锦囊、文策风格、风格卡
  - 若未加载：显示未 loaded 状态并禁止生成；提供“重新加载”
- **可选项（optional）**：前情摘要、上一章片段、额外引用（流光/藏经/推演）
  - 可开关是否注入
  - 显示 token 估算与优先级
- **查看“本次使用材料”摘要**：
  - 显示：模型、作品/章、风格卡、锦囊、细纲、引用、token 粗估、费用估算

### 4.3 快捷 prompt 与输入

- **快捷 prompt**：
  - 点击填入输入框
  - 支持自定义与排序（最终态）
- **输入框行为**：
  - Enter 发送；Shift+Enter 换行
  - 发送按钮在生成中禁用
  - 输入草稿在刷新/跳转后可恢复（最终态）

### 4.4 生成、取消、重试

- **生成**：
  - 入口：发送按钮/快捷按钮“生成全章”
  - 过程：生成中 Loader；流式输出（可选）；记录 tokens/cost
  - 结果：新增一个草稿版本（version++）
- **取消生成**：
  - 入口：生成中“停止”按钮（Square）
  - 结果：保留已生成内容为一个“未完成版本”或丢弃（最终策略统一）
- **重试**：
  - 入口：RefreshCw
  - 行为：沿用相同上下文与参数，生成新版本
- **错误处理**：
  - 模型不可用/Key 错误：提示“打开设置”
  - 上下文过长：提示“精简并重试”（自动裁剪 optional）
  - 成本阈值：数字确认（与全局成本策略一致）

### 4.5 草稿版本管理（快照/历史）

> 现有口径：同章多轮生成可切换历史快照；标注「当前采纳」。快照存储 `localStorage`（`sheng-hui-snapshots.ts`）。

- **版本列表**：
  - 展示：版本号、时间、token、费用、温度
  - 操作：选择版本、删除版本
- **采纳/不采纳**：
  - 每章只能有一个“当前采纳”版本（isAccepted）
  - 采纳后：版本标记为当前采纳，并写入“采纳事件”（可追溯）
- **评分**：
  - 赞/踩用于后续推荐与模型偏好（最终态）

### 4.6 对比（Diff）

- **版本对比**：任意两版草稿对比差异（GitCompare）
- **草稿 vs 正文**：
  - 需要读取写作页当前章正文
  - 对比后可选择合并策略（见 4.7）

### 4.7 写回写作（核心闭环）

写回方式（最终态）：

- **写入 AiPanel 草稿区**：
  - 入口：版本操作“写入草稿”
  - 写回：`liubai:aiPanelDraft...`（与现有口径一致）
  - 结果：跳入 `/work/:id`，AiPanel 显示草稿并可合并
- **直接合并到正文**：
  - 入口：版本操作“合并到正文”
  - 行为：打开写作页的 `AiDraftMergeDialog`，支持三方式：章末追加/光标位插入/替换选区
- **回写推演/文策（可选）**：
  - 若生辉生成了“细纲偏差/改纲建议”，可一键写入推演文策（最终态）
- **存为流光碎片**：
  - 入口：版本操作“存为碎片”
  - 标签自动追加“生辉”，并关联作品/章节

## 5. 数据与写回（读/写/缓存）

### 5.1 数据读取

- 作品与章节结构（用于选择器）
- 推演细纲与节点摘要（outline）
- 锦囊（bible）
- 文策风格与风格卡（style）
- 前情摘要/上一章片段（summary/chapter）
- 本章正文（用于 diff 与合并）

### 5.2 数据写回

- 生辉草稿版本与采纳状态（本地快照 + 可选云端同步，最终态在数据总览统一）
  - 现有实现提示：`localStorage` key `liubai:shengHuiSnapshots:`*
- 写入写作草稿：`liubai:aiPanelDraft...`
- 作为碎片写入流光：`InspirationFragment`
- 作为文策写回推演：WenCeEntry（最终态）

### 5.3 一致性与恢复（最终态要求）

- 刷新页面后仍能看到同章历史版本与“当前采纳”
- 写回写作失败时不丢草稿，允许重试
- 当章节正文已变化时，合并需提示“基于旧正文生成”，并建议先对比再合并

## 6. 跨模块联动（因果链）

### 6.1 推演 → 生辉

- 节点“进入生辉”携带：节点摘要/目标字数/人物地点/冲突点/情绪曲线
- 细纲定稿后“生成全章”直接进入生辉并选中对应节点序列

### 6.2 生辉 → 写作

- 采纳/写入草稿/合并到正文形成写作闭环
- 写作侧栏可回跳到生辉的对应版本（版本号/时间戳）

### 6.3 锦囊 ↔ 生辉

- 生辉读取锦囊作为硬约束（禁写/术语/人物状态）
- 生成中发现设定缺口时，一键跳转锦囊补齐后返回继续生成

### 6.4 流光/藏经 ↔ 生辉

- 引用流光碎片/藏经要点作为 optional 上下文项
- 生成结果可存回流光，形成素材回流

### 6.5 生辉 → 问策

- 对草稿差异/风格问题“一键问策”，自动带入版本文本与上下文摘要

---

## 7. 空态 / 加载态 / 错误态

- **无作品/无章节**：提示去留白或**写作**建立章节，生成按钮禁用并说明原因
- **上下文未加载**：required 项未 loaded 时，显示提示与“重新加载”
- **生成中**：Loader2 + 停止按钮；版本列表锁定以避免误操作
- **错误态**：
  - Key/模型不可用：提示打开设置
  - 费用/阈值：数字确认
  - 上下文过长：精简并重试
  - 网络失败：可重试并保留输入

## 8. 验收与手测脚本（可执行）

### 8.1 基础生成（必过）

1. 进入 `/sheng-hui`，选择作品与章节。
2. 确认 required 上下文项均为 loaded。
3. 使用“续写段落”快捷 prompt 生成草稿，确认产生版本 1。
4. 点击重试生成版本 2，确认版本列表可切换。

### 8.2 采纳与快照（必过）

1. 将版本 1 标记为“当前采纳”，确认标记唯一且持久化（刷新后仍在）。
2. 对版本 2 点赞/点踩，确认评分保存。
3. 删除一个未采纳版本，确认删除需确认且删除后列表更新。

### 8.3 对比与写回（必过）

1. 对比版本 1 vs 版本 2，确认差异可见。
2. “写入写作草稿”并跳转 `/work/:id`，确认 AiPanel 草稿区出现内容。
3. 打开合并弹窗，分别测试：章末追加/光标位插入/替换选区。

### 8.4 错误与恢复（必过）

1. 模拟 required 上下文未加载，确认生成按钮禁用并可重新加载。
2. 模拟网络失败，确认错误提示清晰且可重试，输入不丢失。
