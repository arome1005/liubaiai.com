# 跨模块联动总览（v0 目标态）

> 目标：把“联动因果链”讲到可验收——谁触发、影响谁、数据从哪来、落盘在哪、如何手测。  
> **命名（2026-04-11）**：**「落笔」** = 七模块之一 + §5 能力总域；**「锦囊」** = 原「圣经」设定护栏页（`/work/:id/bible`）；**「写作」** = 正文沉浸页（`/work/:id`）。三者**不可**互相划等号。凡写「写入草稿 / 合并入正文」，落点一律是 **写作页**（右栏 AiPanel）。总表见 `README.md`。

## 1. 七大模块联动地图（最终态）

### 1.1 全局作品上下文（贯穿七模块）

- **来源**：
  - 留白选择作品 / 点击作品卡
  - **写作**页正在编辑的作品（`/work/:id`）
  - 其它模块的作品选择器（推演/问策/生辉等）
- **落点**：
  - Hub 顶栏作品上下文区显示当前作品/章节
  - 各模块默认选中该作品（或优先该作品）
- **保留规则（最终态）**：
  - 记忆最近作品 `lastWorkId`；当路由携带 `preferWorkId` 时优先使用
  - 从任意模块跳转到另一个模块时必须携带当前作品语义（参数或 state）

### 1.2 七大模块的主闭环

- **留白**（作品库）：创作起点与作品管理中心 → **新建成功默认进写作**；亦可进入 **锦囊** / 推演 / 概要 等
- **推演**（规划）：大纲/扫描/文策/三分支 → 产出“可写回草稿/可讨论议题/可执行修复”
- **流光**（素材）：捕获/扩容/整理 → 转入**写作** / 推演 / 问策
- **问策**（对话）：围绕分歧与卡点形成决策与行动项 → 写回推演 / **写作**草稿 / **锦囊** / 流光
- **落笔**（七模块；§5 总域）：数据与能力覆盖锦囊（原圣经）/ 模板 / 笔感 / 全局词典等 → 经装配器进入 **生辉** 与 **写作页** 侧栏 AI（用户编辑锦囊子页在 `/bible`，编辑正文在 `/work/:id`）
- **写作**（正文，非七模块名）：章节与正文编辑 → AI 草稿合并 → 推进概要与进度
- **生辉**（仿写）：按纲生成多版本草稿 → 采纳/对比 → 写回**写作**草稿/正文
- **藏经**（参考）：本地导入/分块/索引/提炼 → RAG 命中进入装配器 → 支撑推演/问策/**写作**

## 2. 四条核心验收链路（终验收必测）

> 这四条链路与现有产品说明书保持一致，但在最终态下要求“v0 UI 对齐 + 全链路可追溯”。

### 链路 1：推演 → 写作（分支写入草稿 → 合并入正文）

```
推演：生成三分支 → 点击「写入草稿」
  → 写作草稿存储（aiPanelDraft）
  → 用户切到**写作**（/work/:id）
  → AiPanel 显示草稿
  → 用户点击「合并入正文」→ diff 弹窗 → 确认
  → 正文更新
  →（可选）概要队列更新 / 推演文策记录一次“采纳”
```

验收点：
- 草稿可追溯来源（来自哪个分支/哪个章节）
- 合并支持：章末追加/光标位插入/替换选区
- 合并失败不丢草稿

### 链路 2：推演 → 问策（分支跟进）

```
推演：三分支结果 → 点击「去问策跟进」
  → 打开问策（/chat）并预填内容
  → 自动关联作品 + 注入设定索引（可选）
  → 用户直接发送继续讨论
  → 输出可一键写回：推演文策 / **写作**侧栏草稿 / **锦囊**条目
```

验收点：
- 预填内容与引用材料可见
- 写回动作存在且可用

### 链路 3：流光 → 写作（素材捕获 → 扩容 → 转入正文）

```
全局速记（Alt+S）→ 保存碎片
  →（可选）去流光扩容 → 生成五段 → 保存为新碎片
  → 流光：选碎片 → 转入写作
    - A：章末追加
    - B：光标位插入
    - C：侧栏草稿合并
  → **写作**保存 → 重进仍在
```

验收点：
- 三方式都可用，且插入位置正确
- 扩容候选可保存/可转入/可问策

### 链路 4：藏经 → 写作 AI（RAG）

```
藏经：导入参考书 → 分块 + 索引
  → **写作**右栏「参考」Tab：开启 RAG + 勾选书目范围
  → 用户触发 AI 生成
  → 装配器执行 top-k 检索（倒排/语义/混合）
  → 命中片段注入 AI 上下文
  → 「本次使用材料」摘要展示命中来源
  → 草稿合并入正文
```

验收点：
- 命中来源可追溯（书名/章节/片段）
- 藏经不上云的约束在 UI 中明确

## 3. 联动矩阵（最终态模板）

> 用于补齐“每个按钮会影响谁”。随着页面完善逐步填充。

| 触发页面 | 触发动作 | 落点页面 | 落点表现 | 联动来源 | 写回位置 | 验收步骤 |
|---|---|---|---|---|---|---|
| 推演 | 三分支→写入草稿 | **写作** | AiPanel 草稿区出现 | 分支结果 + 章节上下文 | aiPanelDraft | 链路1 |
| 推演 | 三分支→去问策 | 问策 | 新会话预填 + 引用材料 | 分支结果 | 会话 messages | 链路2 |
| 流光 | 转入写作 | **写作** | 插入/草稿合并 | 碎片内容 | 正文/草稿 | 链路3 |
| 藏经 | RAG 命中 | **写作** | 本次材料摘要 + 草稿 | 命中片段 | 草稿 | 链路4 |

## 4. 联动的“可追溯”统一要求（最终态）

- **任何 AI 产物**必须能回答：
  - 来自哪个模块（推演/流光/问策/生辉/**写作**）
  - 基于哪些材料（锦囊/概要/RAG/碎片/细纲）
  - 何时生成、用哪个模型、粗估 token 与成本
- **任何写回动作**必须具备：
  - 失败不丢输入
  - 可撤销/可回滚（至少通过快照或草稿历史）
