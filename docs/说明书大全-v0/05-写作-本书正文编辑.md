# 本书正文编辑 — 写作页（v0 目标态）

> **命名**：本文档只写 **「写作」** 页——路由 `**/work/:id`**（`EditorShell`），**不是**顶栏七模块里的「落笔」二字所指的 §5 总域本身。  
> **默认主路径**：留白 **新建作品成功 → 直接进入本页**（见 `01-留白-作品库门户.md`）。  
> **和落笔域的关系**：侧栏 AI（AiPanel）会装配 **锦囊（原圣经）** 与 **§5 落笔域** 其它材料（术语、笔感、模板等）；数据多来自 `/work/:id/bible` 与全局设定，但 **页面名称仍是「写作」**。勿把本页误称为「落笔页」。顶栏模块「落笔」的总说明见 `**08-落笔-能力域与入口.md`**；总定义见 `README.md` 术语表、`design/seven-modules-ui-spec.md` §5 / §9；锦囊分区专论见 `81-锦囊-设定护栏.md`（交叉引用，本文不展开）。  
> UI 口径：**最终 v0 对齐后的写作壳层**（章节树 + 编辑器 + 右侧 AI/设定/工具面板 + 沉浸模式）。  
> 功能口径：**最终全部功能均已实现**（不写“待开发/占位”）。

---

## 1. 页面定位

- **一句话目的**：提供长篇写作的“正文主战场”——稳定编辑、可恢复、可导出，并通过 AI 侧栏把推演/锦囊/藏经/流光的材料变成可采纳的草稿与合并动作。
- **主要用户任务**：
  - 章节/卷管理：新建、切换、重排、删除
  - 正文编辑：自动保存、字数统计、查找替换、排版设置
  - 草稿安全：异常关闭后的草稿恢复、快照回溯
  - 导出与备份：单章/全书导出（TXT/DOCX/MD），以及全量备份 ZIP（在设置页）
  - AI 侧栏：材料勾选 → 生成草稿 → diff 合并（章末追加/光标位插入/替换选区）
  - 沉浸写作：隐藏干扰，保留最关键控制

## 2. 入口与导航

### 2.1 进入本页

- **主入口**：作品库（留白）点击作品卡片主体/“继续写作”。
- **从其它模块进入**：
  - 推演：三分支“写入草稿”后跳入**写作页**合并
  - 流光：碎片“转入写作”三方式跳入
  - 生辉：仿写生成结果写入草稿后跳入
- **深链参数（最终态要求）**：
  - `?zen=1`：进入沉浸写作模式
  - 支持 `state` 携带“要合并的草稿/要插入的片段/要定位的章节与位置”（实现方式不限，验收效果一致）

### 2.2 从本页离开（去哪里）

- 返回作品库：通过顶栏/面包屑（非沉浸）或快捷入口
- 去推演：在章节/卷上“查看大纲/推演”入口
- 去锦囊：右侧“设定”面板直达 `/work/:id/bible`
- 去概要：右侧“概要”或顶栏入口 `/work/:id/summary`
- 去流光：将选区/草稿“一键存为碎片”并可跳转 `/inspiration`

### 2.3 状态保留（最终态要求）

- 记忆项：
  - 当前章节、卷展开状态、章节树滚动位置
  - 编辑器光标位置与选区（至少切换面板/打开合并弹窗时不丢）
  - 右侧面板当前 Tab（AI/设定/参考/工具等）
  - 编辑器宽度、字体与排版偏好
  - AI 侧栏模型选择与“材料勾选”状态
- 返回行为：从推演/问策/锦囊跳回写作页时，仍定位原章节与光标附近上下文。

## 3. 信息架构（v0 目标 UI）

> 写作页使用独立壳层（EditorShell），与 Hub 顶栏不同：更“工具化、更专注”。

### 3.1 顶栏（Editor Top Bar）

从左到右（最终态）：

- **作品信息区**：作品名、当前卷/章；快速切换作品/章节（下拉或面包屑）
- **保存与状态**：
  - 自动保存指示（已保存/保存中/保存失败）
  - 冲突提示（需要重新载入/合并）
- **主要动作**：
  - 导出（单章/全书）
  - 快照/历史（打开快照面板）
  - 查找（本章/全书）
  - 沉浸模式开关（或在更多菜单）
- **右侧工具**：
  - 编辑器宽度/排版设置入口
  - 打开/收起右侧面板按钮

### 3.2 左侧栏：卷/章节树（Chapter Tree）

- **卷管理**：
  - 新建卷、编辑卷标题、删除卷（需要先清空卷内章节）
  - 卷折叠/展开
- **章节列表**：
  - 新建章节
  - 切换章节（切换时不残留上章正文）
  - 拖拽重排（同卷内/跨卷，最终态）
  - 删除章节（高危确认）
- **章节状态**（最终态可选）：
  - 草稿/写作中/完成（与推演节点状态可映射）
  - 字数/更新时间小字显示

### 3.3 中间主栏：正文编辑器（Editor）

- **编辑器内核**：CodeMirror 6
- **编辑体验**：
  - 自动保存（防抖）
  - 字数统计（本章/本书可在状态区展示）
  - 行尾模式/软换行（按设置）
  - 查找替换（本章）与全书搜索入口
- **排版**：
  - 字体族、字号、行高
  - “稿纸底色”（paper tint）
  - 编辑器宽度（720–1600px 拖拽或自适应）

### 3.4 右侧栏：四类面板（Right Rail）

最终态以“Tab”组织（具体命名按 v0）：

- **AI（AiPanel）**：生成草稿、材料勾选、成本控制、合并入口
- **设定（锦囊快照）**：人物/世界观/术语/禁忌/时间线摘要（只读）+ 直达锦囊
- **参考（藏经/RAG）**：检索、收藏、引用、插入/写入草稿
- **工具（快照/导出/风格卡等）**：与写作相关的操作集中

### 3.5 沉浸写作（Zen）

- **进入**：`Alt+Z` 或 URL `?zen=1`
- **表现**：隐藏不必要的导航与边栏；保留最关键的退出/保存/生成入口
- **退出**：`Esc`（或右上角退出按钮）

## 4. 功能清单（角落级）

### 4.1 章节与卷管理

#### 4.1.1 新建卷

- **入口**：章节树顶部“新建卷”
- **过程**：创建卷 → 进入标题编辑 → 回车确认
- **写回**：卷元信息写入作品结构（本地/云端按数据总览）
- **失败**：写回失败提示重试，不丢输入

#### 4.1.2 新建章节

- **入口**：卷内“+ 新建章节”或章节树底部按钮
- **字段**：章节标题（必填），可选摘要/标签（最终态）
- **结果**：切换到新章并聚焦编辑器；字数从 0 开始

#### 4.1.3 切换章节

- **入口**：点击章节项
- **规则**：
  - 切换前若有未保存变更：自动保存或提示（最终态不能静默丢失）
  - 切换后编辑器只显示新章正文，不残留上章

#### 4.1.4 拖拽重排

- **入口**：拖拽章节项
- **行为**：同卷内排序、跨卷移动（最终态）
- **写回**：更新章节顺序与卷归属；保持 `updatedAt` 一致性

#### 4.1.5 删除章节/删除卷

- **入口**：章节/卷的 `...` 菜单
- **确认**：二次确认 + 明确影响范围（快照/草稿恢复/引用）
- **限制**：删除卷需先删除卷内章节（或提供一键级联删除并强化确认）

### 4.2 正文编辑（CodeMirror）

- **输入与编辑**：常规文本输入、撤销/重做、选区操作
- **自动保存**：
  - 防抖写入
  - 顶栏显示“保存中 → 已保存”
  - 失败显示“保存失败，点击重试”
- **字数统计**：
  - 实时统计本章字数
  - 本书累计字数在合适位置展示（顶栏/侧栏）

### 4.3 草稿恢复（draftRecovery）

> 目标：防止意外关闭/崩溃导致当章内容丢失。

- **机制**：每章内容防抖写入 localStorage（以章节为 key）。
- **触发**：再次进入该章节时检测存在未合并草稿恢复内容
- **交互**：
  - 提示条：“检测到未恢复草稿”
  - 按钮：恢复 / 丢弃 / 查看差异（最终态建议）
- **边界**：用户显式丢弃后不再弹；恢复后清理草稿缓存

### 4.4 快照（Snapshots）

> 目标：给“写坏了”提供回到过去的保险栓。

- **触发**：
  - 手动快照：用户点击“快照”
  - 自动快照：保存时自动写入（最终态）
- **展示**：
  - 按章节列出快照列表（时间/字数差异）
  - 支持预览与恢复
- **限制**：每章最多 50 条；超过 7 天清理（最终态与现有口径一致）
- **恢复**：
  - 恢复前二次确认
  - 恢复后生成一条“恢复事件”记录（便于追溯）

### 4.5 查找与全书搜索

- **本章查找替换**：编辑器内 `⌘/Ctrl+F` / 替换
- **全书搜索**：
  - 入口：顶栏搜索或命令面板
  - 输出：匹配列表（章名/命中片段/定位按钮）
  - 点击定位：打开对应章节并高亮命中

### 4.6 导出（单章/全书）

- **单章导出**：
  - TXT / DOCX
  - 入口：顶栏“导出”→ 单章
- **全书导出**：
  - TXT / DOCX / MD
  - 入口：顶栏“导出”→ 全书
- **导出策略（最终态）**：
  - 章节顺序按当前卷/章节树
  - 可选包含：标题层级、分隔符、元信息（作者/时间）
- **失败**：权限/写文件失败提示；提供复制到剪贴板作为降级

### 4.7 沉浸写作（Zen）

- **进入**：`Alt+Z` 或 `?zen=1`
- **行为**：
  - 隐藏非关键 UI（章节树/右侧栏可按 v0 策略决定是否隐藏）
  - 保留退出、保存状态、最关键的生成入口
- **退出**：`Esc`

## 5. AI 侧栏（AiPanel，v0 目标态）

> 目标：把“材料 → 生成 → 草稿 → 合并”做成可控、可解释、可回滚的闭环。

### 5.1 模型与运行模式

- **模型选择**：支持多 Provider（OpenAI/Anthropic/Gemini/豆包/智谱/Kimi/小米/Ollama…）
- **运行模式**：
  - 续写 / 扩写 / 润色 / 自定义 / 抽卡
- **额外要求**：自由文本，可从模板库填入

### 5.2 材料勾选（Context Builder）

可选材料（最终态）：

- 邻章概要
- 本章锦囊字段（目标/禁写/POV/人物状态等）
- 全书锦囊板块（人物/世界观/术语/禁忌/时间线）
- RAG（藏经检索结果）
- 流光碎片（可选）
- 推演节点摘要/三分支/扫描结果摘要（可选）

并提供“本次使用材料”折叠摘要，明确：

- 模型、作品/章、风格卡、锦囊、摘录、标签、RAG、token 粗估

### 5.3 成本控制与高危确认

- token 粗估
- 超阈值（如 20 万）要求数字确认 + 最短停留 + 冷却
- 本会话/本机累计展示

### 5.4 生成与草稿区

- **输出**：流式输出到“草稿区”，与正文分离
- **草稿管理**：
  - 多草稿历史
  - 重命名/复制/删除
  - 一键存为流光碎片（最终态）

### 5.5 草稿合并（AiDraftMergeDialog）

- **打开**：草稿区“合并”按钮
- **合并方式**：
  - 追加到章末
  - 光标位插入
  - 替换选区
- **对比**：diff 弹窗行级对比
- **失败**：合并冲突/章节已更新时提示重新载入并保留草稿

### 5.6 降级重试

- 上下文类错误：提供“精简并重试”
- 模型不可用：切换模型建议 + 打开设置入口

## 6. 数据与写回（读/写/缓存）

### 6.1 读取

- 作品/章节结构与正文
- 锦囊（本章与全书）
- 风格卡与标签侧写
- RAG 检索结果（藏经）
- 推演产物（节点摘要/三分支/扫描结果摘要）
- 流光碎片（可选注入）

### 6.2 写回

- 正文保存（章节内容）
- 草稿恢复缓存（localStorage）
- 快照（localStorage 或等价持久化，最终态策略在数据总览统一）
- AI 草稿写入（会话/本地/云端策略统一）
- 导出文件（本地下载）

### 6.3 并发与冲突（最终态要求）

- `updatedAt` 乐观锁：检测冲突后提示“重新载入本章/合并”
- 冲突时必须保留用户输入与草稿不丢失

## 7. 跨模块联动（因果链）

### 7.1 推演 → 写作

- 三分支“写入草稿” → AiPanel 草稿区 → 合并入正文
- 扫描结果“定位正文/写入修复草稿” → 高亮/草稿合并

### 7.2 流光 → 写作

- 碎片“转入写作”：章末追加/光标插入/草稿合并

### 7.3 生辉 → 写作

- 仿写生成结果进入草稿区；用户确认后合并

### 7.4 写作页 → 锦囊 / 概要

- 写作推进触发概要更新（最终态可自动或手动）
- 设定新增入口（从写作中一键存为锦囊条目）

### 7.5 写作 → 流光 / 问策

- 选区/草稿“一键存为流光碎片”
- 选区“一键问策”进入 `/chat` 并预填

---

## 8. 空态 / 加载态 / 错误态

- **无作品**：提示返回留白创建作品
- **无章节**：引导新建章节并聚焦编辑器
- **加载态**：章节切换 skeleton；AI 生成中流式/可取消
- **保存失败**：显式提示 + 重试；不得静默
- **导出失败**：提示原因并给出降级（复制到剪贴板）

## 9. 验收与手测脚本（可执行）

### 9.1 编辑与保存（必过）

1. 进入 `/work/:id`，新建卷与章节。
2. 输入正文，确认自动保存状态从“保存中→已保存”。
3. 切换章节再切回，确认内容不丢且不串章。

### 9.2 草稿恢复与快照（必过）

1. 在某章输入一段文字后强制刷新页面（模拟意外关闭）。
2. 重新进入该章，确认出现草稿恢复提示；恢复后内容正确。
3. 创建快照并恢复到旧快照，确认恢复成功且有二次确认。

### 9.3 AI 草稿与合并（必过）

1. 在 AiPanel 勾选材料并生成草稿，确认草稿区出现输出。
2. 打开合并弹窗，分别测试：章末追加/光标插入/替换选区。
3. 合并后正文更新且可撤销（最终态）。

### 9.4 导出（必过）

1. 单章导出 TXT/DOCX 成功。
2. 全书导出 TXT/DOCX/MD 成功，章节顺序正确。

### 9.5 沉浸写作（必过）

1. `Alt+Z` 进入沉浸，确认干扰 UI 隐藏且保留退出入口。
2. `Esc` 退出沉浸，返回原状态。
