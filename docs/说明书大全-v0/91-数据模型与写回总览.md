# 数据模型与写回总览（v0 目标态）

> 目标：把“每个模块读什么、写什么、写到哪、怎么同步、怎么回滚、失败怎么处理”集中讲清。  
> 口径：最终态以“可验收、可交接”为准；具体字段以 `src/db/types.ts` 为技术真源。

---

## 1. 术语与实体（统一口径）

- **Work（作品）**：写作项目的顶层容器，包含元信息、标签、状态、封面与进度指针。
- **Volume（卷）**：作品内的章节分组与顺序容器。
- **Chapter（章节）**：正文与概要的主要承载实体。
- **ChapterSnapshot（正文快照）**：章节内容的历史版本，用于回滚。
- **ChapterBible（本章约束）**：本章目标/禁写/POV/场景立场/人物状态等，作为写作与扫描的硬约束。
- **锦囊条目（作品级设定）**：
  - 人物（BibleCharacter）
  - 世界观（BibleWorldEntry）
  - 伏笔（BibleForeshadow）
  - 时间线（BibleTimelineEvent）
  - 术语（BibleGlossaryTerm）
  - 风格卡（WorkStyleCard）
  - 提示词模板（WritingPromptTemplate）
  - 笔感样本（WritingStyleSample）
- **推演产物（作品级）**：
  - 大纲节点（卷/章/场景）及其字段
  - 文策（决策/修订/AI建议/灵感/里程碑）
  - 扫描结果（规则/AI）与忽略状态
  - 三分支结果
  - 地点与地点事件（LogicPlaceNode/Event）
- **流光碎片**：InspirationFragment（可关联作品/集合/标签）。
- **问策会话**：WenceChatSession（messages + includeSettingIndex + workId?）。
- **生辉快照**：同章多版本草稿（版本、采纳、评分、tokens/cost）。
- **藏经（参考库）**：
  - 书目元数据、分块、章节索引、倒排索引、摘录、标签、提炼要点
  - 硬约束：始终本地 IndexedDB，不上云

## 2. 存储与同步策略（最终态）

### 2.1 三轨存储

- **云端（Supabase）**：写作域数据（作品/卷/章/锦囊/推演地点/流光/问策…）在开启云同步时进入云端表，受 RLS 约束。
- **本地（IndexedDB）**：作为本地优先的主缓存层；未配置云或未登录时承载全部写作域数据。
- **本地轻量持久化（localStorage/sessionStorage）**：用于 UI 状态与跨页草稿/快照等小数据。

### 2.2 Hybrid 最终策略（与现有口径一致）

- 未配 Supabase 环境变量：纯 IndexedDB（全本地）
- 配齐并登录：Hybrid（写作/锦囊/风格卡/流光/问策写入 Supabase；藏经仍在 IndexedDB）
- 登出或未配：自动回退纯 IndexedDB

### 2.3 藏经不上云（硬约束）

理由（最终态必须在 UI/文档明确）：

- 版权与隐私风险
- 文件体积与索引成本
- 离线可用与性能考虑

## 4. 写回规则（最终态统一规范）

### 4.1 “草稿区优先”原则（AI 产物统一入口）

- 推演三分支 / 生辉仿写 / **写作页** AI（AiPanel）/ 流光扩容 / 问策建议  
**一律先进入草稿区或候选区**，由用户确认后再写回正文/设定/大纲。

### 4.2 写回的三种类型

- **直接写回（低风险）**：
  - 标签/收藏/排序偏好等
- **确认写回（中风险）**：
  - 覆盖概要、应用到大纲字段、写入锦囊条目
- **高危写回（高风险）**：
  - 删除作品/章节/书籍
  - 全量恢复
  - 大额 token 调用

### 4.3 冲突与并发

- 章节正文用 `updatedAt` 乐观锁：冲突提示“重新载入/合并”
- AI 草稿与快照必须独立于正文保存：即使正文冲突，草稿仍可被保留并重新合并

### 4.4 删除/恢复策略

- 删除必须二次确认，并明确级联范围
- 关键内容支持“快照回滚”或“恢复版本”
- 全量恢复必须强确认（输入确认文案/数字确认）

## 5. 失败策略（最终态）

### 5.1 网络失败

- 云端写回失败：回退到本地队列并提示“待同步”（最终态）
- 允许重试；不得静默失败

### 5.2 AI 错误

- Key/模型不可用：提示并直达设置
- 上下文过长：提供“精简并重试”（自动裁剪 optional）
- 成本阈值：数字确认

### 5.3 索引失败（藏经）

- 导入/索引失败：给出失败报告，允许重建索引或清理半成品
- 提供降级检索模式（仅关键词/仅语义）

## 6. 端到端数据流示例（最终态）

### 6.1 推演三分支写入草稿 → 合并入正文

- 生成分支 → 写入 aiPanelDraft → **写作页** AiPanel 读取 → diff 合并 → chapter.content 更新 → chapter_snapshot 写入

### 6.2 藏经 RAG → 写作页生成

- referenceChunks/tokenPostings 检索 top-k → assemble-context 注入 → 草稿生成 → 合并入正文

### 6.3 问策写回锦囊条目

- 对话产出建议 → 一键写入 BibleWorldEntry/GlossaryTerm → 推演扫描与 **锦囊 / 写作侧栏** 注入立即生效
