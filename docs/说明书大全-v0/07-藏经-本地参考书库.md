# 藏经 — 本地参考书库（v0 目标态）

> 路由：`/reference`  
> UI 口径：**最终 v0 对齐后的藏经工作台**（书库列表 + 搜索筛选 + 导入/导出 + 提炼与收藏 + RAG 引用）。  
> 功能口径：**最终全部功能均已实现**（不写“待开发/占位”）。  
> 硬约束：参考库（藏经）**始终本地**不上云（与产品原则一致）。

---

## 1. 页面定位

- **一句话目的**：把参考书/资料导入本地、分块索引，并提供检索与“提炼要点”，用于推演 / 问策 / **写作页**（AiPanel）等的引用与 RAG 注入。
- **主要用户任务**：
  - 导入参考书（txt/pdf/docx 等）并自动分块+索引
  - 书库管理：标签、收藏、删除、批量导出
  - 搜索与筛选：按标签/收藏/阅读进度/字数/最近阅读排序
  - 提炼要点：把书中内容提炼为人物/世界观/情节/技法等结构化条目
  - RAG 检索：在 **写作页** / 问策 / 推演中开启 RAG，命中条目进入“装配器”
  - 重建索引：当倒排索引损坏或升级后，支持一键重建

## 2. 入口与导航

### 2.1 进入本页

- **主入口**：Hub 顶栏“藏经”。
- **从其它模块进入**：
  - **写作页**右侧“参考 / RAG”Tab → “管理藏经”跳转
  - 推演右侧“参考”面板 → “浏览藏经”跳转
  - 问策右侧“引用材料” → “从藏经添加”跳转

### 2.2 从本页离开（去哪里）

- **去写作页**：对某本书 / 某条摘录“一键引用到草稿 / 插入正文”
- **去推演**：对提炼要点“一键应用到大纲/写入文策”
- **去问策**：将摘录/要点作为引用材料预填到会话

### 2.3 状态保留（最终态要求）

- 记忆项：视图（网格/列表）、搜索词、标签筛选、是否仅看收藏、排序方式、当前选中书目。
- 返回行为：从 **写作页** / 推演 / 问策跳回藏经时，保持筛选与滚动位置。

## 3. 信息架构（v0 目标 UI）

> v0 `cangjing-module.tsx` 的骨架是“顶部模块头 + 工具条（搜索/筛选/排序/收藏/视图）+ 统计卡 + 书库列表（网格/列表）”。

### 3.1 模块头（Module Header）

- 标题：藏经阁
- 说明：本地参考书库（强调不上云/用途）
- 右侧主操作：
  - **批量导出**（outline 按钮）
  - **导入书籍**（主按钮）

### 3.2 工具条（Search & Filters）

- **搜索框**：书名/作者/标签关键字；右侧清空按钮（X）
- **标签筛选**（Dropdown）：
  - “全部” + 常用标签集合
  - 选择后立即过滤
- **排序**（Dropdown）：
  - 最近阅读（recent）
  - 字数（words）
  - 进度（progress）
- **仅看收藏**（Star/Toggle）
- **视图切换**：网格（Grid3X3）/列表（List）

### 3.3 统计区（Stats Cards）

最终态建议至少包含：
- 总字数（全库）
- 已提炼条目数（全库）
- 书目数（可选）
- 本地存储占用（可选，强调 HardDrive）

### 3.4 书库列表（Grid / List）

#### 3.4.1 网格卡片（Book Card）

每本书卡片包含：
- 封面色块（coverColor）或封面图（最终态可选）
- 书名、作者
- 标签 chips
- 章节数/字数
- 阅读进度条（Progress）
- 提炼数（extractedCount）
- 收藏星标
- `...` 菜单：编辑/删除/导出/查看详情（最终态）

#### 3.4.2 列表视图（Book Row）

信息等价但横向展开，便于批量勾选与导出。

### 3.5 书籍详情 / 提炼 / 摘录（弹层或内页，最终态）

在 v0 工作台内，书籍卡片点击进入“书籍详情”：

- 章节目录（ReferenceChapterHead）
- 正文分块浏览（referenceChunks）
- 摘录列表（referenceExcerpts）
- 标签管理（referenceTags）
- 提炼面板（结构化要点：人物/世界观/情节/技法）
- “应用到推演 / 问策 / **写作**”动作区

## 4. 功能清单（角落级）

### 4.1 导入书籍（Upload）

> 来自现有产品说明书：支持 `.txt` / `.pdf` / `.docx`；自动分块 + 索引；批量导入有进度。

- **入口**：模块头“导入书籍”
- **支持格式**：txt / pdf / docx（最终态可扩展 md/epub）
- **流程**：
  1. 选择文件（可多选批量）
  2. 解析文本（pdf/docx 提取正文）
  3. 自动分章（若可识别章节标题）→ 生成 `ReferenceChapterHead`
  4. 分块（chunking）→ 写入 `referenceChunks`
  5. 建索引（倒排 postings + 可选 embedding）→ 写入 `referenceTokenPostings`（与 embedding 缓存表）
  6. 完成后，书目出现在列表并显示进度/提炼数为 0
- **进度与可取消**：
  - 每本书显示进度条与当前阶段（解析/分块/索引）
  - 支持取消单本导入（取消后不产生半成品或可一键清理）
- **失败与提示**：
  - 解析失败：提示原因（加密 pdf/损坏 docx）与替代方案（转 txt）
  - 超大文件：提示分卷导入或先裁剪
  - 存储空间不足：提示清理与导出备份

### 4.2 搜索（Search）

- **入口**：工具条搜索框
- **范围**：书名/作者/标签
- **行为**：即时过滤书目列表；Esc/清空按钮清空

> 深度检索（全文 RAG）在“书籍详情 / 检索模式”里提供（最终态），并与 **写作页** / 问策的 RAG 共用检索器口径。

### 4.3 标签筛选（Tag Filter）

- **入口**：筛选下拉（Filter）
- **行为**：选中标签 → 仅显示含该标签书目
- **回显**：按钮文本显示当前标签；可一键回到“全部”

### 4.4 排序（Sort）

- **入口**：排序下拉（SortAsc）
- **选项**：
  - 最近阅读（lastRead）
  - 字数（words）
  - 进度（readProgress）
- **反馈**：不改变筛选/搜索，仅改变顺序

### 4.5 收藏（Favorites）

- **入口**：星标按钮 + “仅看收藏”开关
- **写回**：更新 isFavorite（本地）
- **联动**：推演/问策引用时可优先展示收藏书目（最终态）

### 4.6 批量导出（Download）

> 来自现有产品说明书：选中书目导出为 ZIP。

- **入口**：模块头“批量导出”
- **选择方式**：
  - 列表视图勾选多本
  - 或导出“当前筛选结果”
- **导出内容（最终态）**：
  - 书目元数据（title/author/tags/进度/评分）
  - 章节索引（ReferenceChapterHead）
  - 分块正文（referenceChunks）
  - 摘录与标签（referenceExcerpts/referenceTags）
  - 索引（referenceTokenPostings）与 embedding 缓存（可选）
- **失败**：浏览器下载失败时提示重试；提供分包导出作为降级

### 4.7 删除书籍（Trash）

- **入口**：书籍卡片 `...` 菜单“删除”
- **确认**：二次确认，明确会删除：
  - 书目元数据
  - 分块与索引
  - 摘录/标签/提炼要点
- **联动**：被 **写作页** / 推演 / 问策引用的书目删除后，引用处应显示“来源已移除”的降级提示（不导致崩溃）

### 4.8 提炼要点（Extract）

最终态把“引用 → 提炼 → 应用”做成闭环：

- **入口**：书籍详情“提炼”按钮；或对某段摘录“提炼为要点”
- **提炼类型**：
  - 人物（character）
  - 世界观（worldview）
  - 情节（plot）
  - 技法（technique）
- **输出**：
  - 结构化要点卡片（可编辑标题/内容/标签/来源）
  - 可收藏、可删除、可检索
- **写回**：保存到本地提炼表（或 excerpts 扩展字段），并计入 extractedCount

### 4.9 RAG 检索与“装配器”（Assembler）

> 来自现有产品说明书：写作 AI 侧栏开启 RAG 时 top-k 命中进入装配器。

- **入口**：
  - **写作页** AiPanel：勾选“RAG”并选择参考书范围
  - 问策：右侧“引用材料”→ 从藏经添加
  - 推演：右侧“参考”→ 关联藏经条目
- **检索模式**（最终态，记忆于 `liubai:referenceSearchMode`）：
  - 关键词倒排（postings）
  - 语义检索（embedding）
  - 混合检索（最终态推荐）
- **输出**：
  - top-k 命中列表：命中片段、来源书名、章节、位置
  - 一键“加入本次材料”（进入 AiPanel / 问策引用 / 推演参考）
- **写回**：
  - 不直接写正文；进入草稿区/文策/节点字段后由用户确认写回

### 4.10 重建索引（Rebuild）

> 来自现有产品说明书：倒排索引损坏时重建。

- **入口**：工具条/设置菜单“重建索引”
- **过程**：对选中书目或全库重建 postings/embedding
- **反馈**：进度条；完成提示；失败提示与可重试

## 5. 数据与写回（读/写/缓存）

### 5.1 数据表（与现有产品说明书对齐）

- `referenceLibrary`：参考书元数据
- `referenceChunks`：正文分块
- `referenceChapterHeads`：章节标题索引
- `referenceTokenPostings`：倒排索引
- `referenceExcerpts`：摘录
- `referenceTags`：标签
- `referenceExcerptTags`：摘录-标签关联

### 5.2 读取

- 书库列表（元数据 + 统计）
- 书籍详情（章节索引、分块、摘录、提炼要点）
- 检索器（postings/embedding）

### 5.3 写回

- 导入：新增书目 + 分块 + 索引
- 更新：标签、收藏、进度、评分、提炼要点
- 删除：级联删除书目相关全部数据
- 导出：从本地聚合打包 ZIP

### 5.4 缓存与一致性（最终态要求）

- 大文件导入与索引必须可恢复（中断后可继续或可清理）
- 检索模式与偏好持久化（`liubai:referenceSearchMode`）
- embedding 缓存要有版本号/失效策略（升级后触发重建）

## 6. 跨模块联动（因果链）

### 6.1 藏经 → 写作页（RAG）

- **触发点**：AiPanel 勾选 RAG 并选择书目范围
- **落点表现**：
  - 命中片段进入“装配器/本次使用材料”
  - 生成结果进入草稿区并可合并
- **来源**：倒排/embedding 命中片段 + 章节定位信息

### 6.2 藏经 → 推演（参考）

- **触发点**：推演右侧“参考”面板关联书籍/应用要点
- **落点表现**：
  - 要点写入大纲节点字段/标签/冲突点
  - 或写入文策记录（含来源书名）

### 6.3 藏经 → 问策（引用材料）

- **触发点**：问策添加引用材料（摘录/要点）
- **落点表现**：系统消息条显示已加载引用；回答中可引用来源并生成行动项

### 6.4 其它页 → 藏经

- 从流光 / **写作页** 保存“摘录”进入藏经摘录库（最终态）
- 从推演/问策把“引用材料”回填为藏经摘录（便于复用）

---

## 7. 空态 / 加载态 / 错误态

- **空态**：无书 → 引导导入；展示格式支持与“不上云”提示
- **导入中**：每本书进度条 + 阶段；可取消
- **错误态**：
  - 解析失败（pdf/docx）
  - 索引失败/损坏（提示重建）
  - 存储不足
- **自救**：
  - 导出备份后清理
  - 重建索引
  - 降级检索（仅关键词/仅语义）

## 8. 验收与手测脚本（可执行）

### 8.1 导入与索引（必过）

1. 进入 `/reference`，导入一本文本与一本 pdf/docx。
2. 确认显示导入进度并完成后出现在书库列表。
3. 打开书籍详情，确认章节索引与分块可浏览。

### 8.2 搜索/筛选/排序（必过）

1. 搜索书名/作者/标签均可命中。
2. 标签筛选与仅看收藏开关可用。
3. 排序三种方式可用且顺序正确。

### 8.3 提炼与应用（必过）

1. 对某段内容提炼为“技法/世界观”等要点，确认 extractedCount 增加。
2. 在推演右侧“参考”应用要点到某节点，确认节点字段/文策记录变化且可追溯来源。

### 8.4 RAG 联动（必过）

1. 在 **写作页** AiPanel 勾选 RAG 并选择书目范围，确认 top-k 命中进入装配器。
2. 生成草稿并合并到正文，确认引用内容可追溯来源书目。

### 8.5 导出与删除（必过）

1. 批量导出选中书目为 ZIP 成功。
2. 删除一本被引用的书，确认引用处降级提示存在且不崩溃。
