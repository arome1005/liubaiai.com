# 落笔 — 能力域与入口（v0 目标态）

> **模块名**：顶栏七模块之一 **「落笔」**  
> **产品语义**：**§5 能力总域**——决定「这本书写出来像不像这本书」的一整套能力（提示词库、结构化设定、指令模板、笔感样本、全局词典等）。权威定义见 `design/seven-modules-ui-spec.md` 第 5 节。  
> **路由说明**：落笔 **不是** 单一能力点，而是 **一层能力 + 多个入口**。**独立首页**：`**/luobi`**（创作工具箱：提示词 / 锦囊 / 世界观 / 风格卡 / 词典等卡片，无需先选中作品 即可进入）。本书编辑向：进入具体作品后，锦囊子页为 `**/work/:id/bible`**，写作页为 `**/work/:id**`。  
> **与另两篇的分工**：**本书锦囊（设定护栏）** 的「分区 / 条目 / 保存」见 `81-锦囊-设定护栏.md`；**全屏正文编辑** 见 `05-写作-本书正文编辑.md`。本篇写 **模块级** 的定位、能力块、入口与联动，避免与 81、05 重复写角落级交互。

---

## 1. 页面定位（模块级）

- **一句话目的**：为单部作品沉淀 **可检索、可注入、可参与扫描与生成** 的写作约束与风格资产，让推演 / 生辉 / **写作页** AiPanel 在装配上下文时「有据可依」，避免 AI 乱人设、乱术语、乱语气。
- **主要用户任务**：
  - 维护 **结构化设定**（UI「锦囊」/ 原 *story bible*）：人物、世界观、术语、禁忌、时间线等。
  - 维护 **指令模板**、**笔感样本**、**全局词典**（与锦囊互补：前者偏「怎么说」，锦囊条目偏「是什么」）。
  - 从顶栏或作品子导航 **进入落笔域工作台**，在锦囊子页与（最终态）其它子路由之间切换。
  - 与 **写作页**、**生辉**、**推演** 形成写回与引用闭环（细则见第 6 节与 `90-跨模块联动总览.md`）。

---

## 2. 必须先分清的三个名字


| 称呼     | 是什么                                    | 典型路由 / 界面                                                 |
| ------ | -------------------------------------- | --------------------------------------------------------- |
| **落笔** | 七模块名 + §5 **整域能力**                     | 顶栏「落笔」→ `**/luobi`**（工具箱）；具体作品数据在 `**/work/:id/bible`** 等 |
| **锦囊** | 仅 **原「圣经」** 设定护栏在 UI 上的名称，**≠**「落笔」同义词 | `**/work/:id/bible`**                                     |
| **写作** | 本书 **正文** 沉浸编辑                         | `**/work/:id`**（留白新建成功默认进入）                               |


**禁止混称**：不要把 **写作页** `/work/:id` 叫作「落笔页」；不要把「落笔」整体叫作「锦囊」。

## 3. 入口与导航

### 3.1 进入落笔域

- **主入口**：Hub 顶栏 **「落笔」** → `**/luobi`**（创作工具箱；**不依赖** 当前是否选中作品）。
- **从工具箱点入本书能力**：若本地有 **最近一次作品** `liubai:lastWorkId`，卡片链到对应 `**/work/:id/bible?tab=…`** 或 `**/work/:id`**（风格卡）；若无记录，引导前往 **作品库** 新建或打开作品。
- **直达锦囊编辑**：仍可通过作品库卡片、「继续编辑」等进入 `**/work/:id/bible`**。
- **从作品库**：作品卡快捷入口「锦囊」等同进入该子页（作品域一致）。
- **从写作页**：右侧「设定 / 打开锦囊」等入口进入同一锦囊子域（见 `05`）。

### 3.2 作品子导航（与 AppShell 一致）

在同一作品下，落笔域与下列子页并列（具体文案以 v0 为准）：

- **写作** `/work/:id` — 正文主战场（**非**落笔模块名）。
- **概要** `/work/:id/summary`
- **锦囊** `/work/:id/bible` — 落笔域内 **结构化设定** 主阵地。
- **作品库** — 回留白。

### 3.3 离开与返回

- 去 **写作页**：继续写正文；锦囊侧编辑的条目可在 **写作页** AiPanel 中勾选注入。
- 去 **推演 / 生辉 / 问策**：携带 `workId`；扫描或仿写读取锦囊与词典等落笔域数据。
- 返回行为：从推演「打开锦囊条目」等深链应能 **定位到锦囊内具体分区 / 条目**（见 `81`）。

### 3.4 最终态可扩展路由（规格层）

规范允许将模板库、笔感、词典等拆为 **同作品下子路由**；v0 若仍集中在锦囊页内以 Tab / 分区呈现，以 v0 UI 为准，但 **模块名仍统一称「落笔」**。

---

## 4. 信息架构（模块级）

落笔域在信息架构上承担 **「作品级资产层」**：

- **锦囊子页**（`/bible`）：分区导航 + 条目列表 + 详情编辑（详见 `81`）。
- **模板 / 笔感 / 词典**（§5）：在 v0 中可能作为锦囊页内分区，或独立子页；装配器从统一数据源读取。
- **与写作页的分界**：**正文与章节结构** 在 **写作页**；**设定与风格资产** 在 **落笔域**（锦囊 + 扩展表）。

## 5. 能力块（与 spec §5 对齐）


| 能力块           | 用户可感知价值                                    |
| ------------- | ------------------------------------------ |
| **结构化设定（锦囊）** | 人物 / 世界观 / 术语 / 禁忌 / 时间线等可检索、可注入、可参与一致性扫描。 |
| **指令模板**      | 可复用的高质量 Prompt，插入或供生辉快捷使用。                 |
| **笔感样本**      | 本书或名家片段作风格锚点，约束语气与节奏。                      |
| **全局词典**      | 专有名词与译名一致，减少胡编术语。                          |


**可实现性提示**：锦囊骨架已有；模板 / 笔感 / 词典与生辉、写作侧栏的 **完整闭环** 可为增量（见 `design/seven-modules-ui-spec.md` §5 可实现性）。

---

## 6. 跨模块联动（摘要）

- **落笔域 → 写作页**：AiPanel 勾选锦囊条目、术语、笔感等 → 装配 → 草稿 → 合并正文（`05`、`91`）。
- **落笔域 → 生辉**：仿写硬约束材料（细纲、锦囊、风格卡、词典等）（`06`）。
- **落笔域 → 推演**：规则 / AI 扫描读取禁忌、术语、时间线；扫描结果可跳回锦囊条目（`02`）。
- **问策 → 锦囊**：建议一键写入设定条目 / 术语 / 时间线（`04`）。
- **设置**：「落笔」相关默认模型与门控影响本域生成链路；编辑器排版项作用于 **写作页**（`80`）。

更细的因果链见 `**90-跨模块联动总览.md`**、数据写回见 `**91-数据模型与写回总览.md`**。

---

## 7. 验收要点（模块级）

- 顶栏「落笔」在有作品时可进入 `**/work/:id/bible**`，且作品上下文与作品卡一致。
- 区分清晰：**新建作品默认进写作页**；进「落笔」多进锦囊子页；不在文案里把三者混称。
- 修改锦囊条目后，推演扫描与写作页注入能反映最新设定（允许异步队列的最终态策略）。
