# 留白写作：v0 说明书大全对照盘点报告（2026-04-16）

> **目的**：基于仓库内 `docs/说明书大全-v0/`（v0 目标态说明书）与当前代码实现（`src/pages/`* 等），评估：
>
> - **完成了多少**（以“功能可用/链路跑通”与“v0 UI/IA 最终态”两种口径分别估算）
> - **还差什么**（按模块与优先级列出缺口）
> - **是否存在超额完成/偏离说明书**（对后续范围管理有帮助）
>
> **约束**：本报告为静态盘点；不涉及修改代码。

---

## 1. 总体结论（两种口径）

### 1.1 功能口径（不强制 v0 UI/IA）

- **总体完成度**：约 **75% ± 5%**
- **判断依据**：七大模块核心闭环多数已可用；写作/藏经/锦囊/设置等完成度较高；推演/问策仍有“工作台化 + 写回体系化 + 可追溯资产化”的明显缺口。

### 1.2 最终态口径（包含 v0 UI/IA 对齐）

- **总体完成度**：约 **60% ± 5%**
- **原因**：说明书大全明确以“最终 v0 对齐 UI + 全功能最终态”为口径描述，包含大量角落级交互、布局骨架（左树/中栏/右栏、多 Tab）与跨模块深链定位等要求；当前实现虽有较多功能，但部分模块尚未完全以 v0 信息架构呈现。

## 3. 按模块对照（完成度 + 缺口 + 超额/偏离）

> 分级口径：
>
> - **已完成**：说明书关键能力基本齐备，且链路可跑通
> - **基本可用**：核心能力可用，但与说明书最终态存在明显角落级/UI/联动缺口
> - **部分完成**：能力要素存在，但离说明书最终态工作台仍有较大距离

### 3.1 留白（/library）作品库门户 —— **基本可用（≈80%）**

- **已完成**：
  - 新建作品、导入作品、网格/列表、搜索、排序、封面、标签、进入写作/锦囊/概要入口等。
- **主要缺口（对照 `01-留白-作品库门户.md`）**：
  - “统计区（四张统计卡）+ 筛选联动 + 回显”的最终态一致性（当前更多偏“功能齐”，但 UI/IA 未必完全按说明书表现）。
  - 批量操作、归档/回收站的最终态交互与文案一致性（说明书对“影响范围/可恢复策略”的表达更严格）。
- **超额/增强**：
  - 工程稳定性处理（例如导入 busy 复位、避免大书架 refresh 卡住）属于说明书未必逐条列出但产品质量加分项。

### 3.2 写作（/work/:id）本书正文编辑 —— **已完成度高（≈85%）**

- **已完成**（对照 `05-写作-本书正文编辑.md` 的关键点）：
  - 卷/章管理、拖拽重排、自动保存、草稿恢复、快照历史（50 条 + 7 天）、查找替换、全书搜索、导出 txt/docx/md。
  - 沉浸（zen）。
  - AiPanel 草稿区 + diff 合并（章末追加/光标插入/替换选区）。
  - “本次使用材料”摘要、RAG 注入能力等。
- **主要缺口**：
  - 右侧“AI/设定/参考/工具”四类面板在最终态的 **Tab 组织、命名与入口位置**（功能要素很多已具备，但是否与说明书描述的最终态一致仍需按验收脚本逐条核对）。
  - 全局命令面板（`⌘/Ctrl+K`）是否已达到说明书“跨模块搜索与跳转”的最终态口径（需要按说明书脚本手测确认）。

### 3.3 锦囊（/work/:id/bible）设定护栏 —— **基本可用（≈80%）**

- **已完成**（对照 `81-锦囊-设定护栏.md`）：
  - 人物/世界观/伏笔/时间线/术语表/风格卡/提示词模板/笔感样本的 CRUD 与基本排序能力。
- **主要缺口**：
  - “条目级深链定位（从扫描/建议跳入定位到条目）”“批量操作”“更强的筛选/状态管理”等最终态细节。
  - 条目级“写入 AI 注入材料预览”与“可追溯”在 UI 上的最终态呈现。

### 3.4 概要（/work/:id/summary）总结概览 —— **基本可用（≈75%）**

- **已完成**：
  - 概要总览、手工编辑保存、覆盖范围/更新时间/过期提示等。
- **主要缺口（对照 `82-概要-总结概览.md`）**：
  - “AI 生成概要：本章/最近 N 章/全书 + 可取消 + 先草稿后覆盖确认 + 可追溯来源”的完整最终态闭环。
  - 概要版本历史（说明书写为可选最终态项）。

### 3.5 推演（/logic）大纲与逻辑规划 —— **部分完成（≈55%）**

- **已完成（链路级能力）**：
  - 三分支预测；规则/AI 扫描相关能力；导图（ReactFlow）；写入写作草稿（`writeAiPanelDraft`）；去问策跟进（`writeWenceHandoff`）；引用藏经摘录等。
- **主要缺口（对照 `02-推演-大纲与逻辑规划.md`）**：
  - v0 最终态“左大纲树/中三视图（大纲/导图/文策）/右三 Tab（详情/AI 对话/参考）”的完整工作台形态与角落级能力（节点状态、定稿进度、文策日志体系、参考要点应用到节点/文策等）。
  - 推演资产“可追溯沉淀”：扫描结果/分支/文策如何成为可检索、可复用、可写回的结构化资产（说明书要求更强）。

### 3.6 流光（/inspiration + 全局速记）灵感碎片管理 —— **基本可用（≈75%）**

- **已完成**：
  - 全局速记、碎片管理、AI 五段扩容、转入写作（handoff），以及备份/导入导出相关工程能力。
- **主要缺口（对照 `03-流光-灵感碎片管理.md`）**：
  - 文件夹/集合更完整的 v0 工作台结构、更多视图（瀑布流等）、批量动作、联动到问策/推演的最终态入口与一致性。
- **超额/增强**：
  - demo seed、批量粘贴/扩容 handoff 等属于说明书之外的增强项。

### 3.7 问策（/chat）策略对话咨询 —— **部分完成（≈60%）**

- **已完成**：
  - 多会话、策略类型、流式对话、注入设定索引相关能力，以及一定的云同步能力。
- **主要缺口（对照 `04-问策-策略对话咨询.md`）**：
  - 右侧 Tabs（上下文/模板/技法卡）的完整最终态；以及“从问策一键写回到写作草稿/推演文策/锦囊条目/流光碎片”的体系化动作呈现与可追溯记录。
  - “技法卡”作为资产（可检索/可复用/可应用）的最终态闭环。

### 3.8 生辉（/sheng-hui）AI 仿写生成 —— **基本可用（≈70%）**

- **已完成**：
  - 上下文装配、生成、快照/采纳、写回写作草稿；以及锦囊/风格卡/RAG 等注入相关逻辑与门控。
- **主要缺口（对照 `06-生辉-AI仿写生成.md`）**：
  - “多版本对比 Diff 工作台式 UI”“右侧版本/对比/设置面板”以及 v0 目标 IA 的完整对齐。

### 3.9 藏经（/reference）本地参考书库 —— **基本可用（≈80%）**

- **已完成**：
  - txt/pdf/docx 导入、本地解析、分块索引、搜索、摘录、标签、提炼结果、批量导出、重建索引、阅读器按需加载等。
  - “不上云”约束在 UI/流程中已强化（近期也增强了导入期间提示）。
- **主要缺口（对照 `07-藏经-本地参考书库.md`）**：
  - 说明书提到的“提炼要点结构化卡片（人物/世界观/情节/技法）+ 应用到推演/问策/写作”的最终态分区与联动按钮体系。
  - “取消单本导入/不产生半成品”的严格一致性策略（说明书要求更强）。

### 3.10 落笔（/luobi + /prompts 等）能力域 —— **相对说明书：超额完成**

- **已完成/超额**（对照 `08-落笔-能力域与入口.md`）：
  - `/luobi` 工具箱入口与“落笔是能力域入口、不依赖选中作品”的定义高度一致。
  - `/prompts` 全局提示词库具备较完整的“草稿/提交/审核/精选/收藏/装配”链路。
  - 书名/简介/NPC 命名生成器属于增值能力（说明书未必强制，但符合“落笔域工具箱”的产品形态）。
- **主要缺口**：
  - 若严格按最终态：落笔域“全局词典”等资产层的更完整 UI/数据体系（目前已有术语表/样本/模板，但全局域与本书域的最终态呈现仍可继续加强）。

### 3.11 设置（/settings）系统配置中心 —— **已完成度高（≈85%）**

- **已完成**（对照 `80-设置-系统配置中心.md`）：
  - AI 厂商配置弹窗、默认模型与阈值、成本统计、编辑器排版、快捷键、主题、备份导出/合并导入/恢复、存储配额、参考库维护、隐私与法律入口等。
- **主要缺口**：
  - 按模块统计趋势图/预算预警等更“最终态”数据可视化（说明书里有，但可作为后期增强）。

## 4. 最关键缺口 Top 10（按说明书“终验收/跨模块联动”优先级）

1. **推演 v0 工作台完整落地**：左树/中三视图/右三 Tab、节点状态与定稿进度、文策日志体系、参考要点“应用到节点/文策”的闭环。
2. **问策写回中心化**：模板胶囊、技法卡资产、引用材料管理，以及“一键写回到写作草稿/锦囊/推演文策/流光”的可追溯动作体系。
3. **跨模块深链定位**：扫描/建议/引用能直接定位到锦囊具体条目、写作具体章节与位置（说明书对“定位与回显”要求很高）。
4. **AI 产物可追溯一致化**：来源模块、材料清单、模型与粗估成本、生成时间、写回记录（当前基础设施较强，但需要做到“处处一致、处处可见”）。
5. **藏经提炼结构化**：从“提炼结果文本”升级为说明书定义的人物/世界观/情节/技法要点卡，并具备“应用到推演/问策/写作”的落点动作。
6. **留白统计区/批量操作最终态**：统计卡、筛选联动、批量管理与说明书一致。
7. **概要 AI 生成最终态交互**：先草稿后覆盖确认、多范围、可取消、可追溯来源。
8. **命令面板终验收口径**：达到说明书“跨模块搜索与跳转”的最终态（需要按脚本手测逐条验收）。
9. **高危操作一致策略**：全站删除/清空/覆盖统一确认交互、提示语气与影响范围说明（近期已在藏经页开始统一）。
10. **v0 UI 对齐收口**：说明书硬性要求“UI 外观与 IA 以 v0 为准”，上线前会是高风险门槛。

## 5. 超额完成 / 偏离说明书的点（范围管理用）

### 5.1 超额完成（正向）

- **Supabase Hybrid + 云同步工程能力**：说明书把它当“最终态策略”，当前实现已具备相当部分工程链路。
- **全局提示词库 `/prompts` 的审核流**：在说明书“落笔能力域”之外，做出了更完整的“提交→审核→精选”链路。
- **藏经 AI 聊天提炼 + 推送提示词库**：属于说明书之外的增强链路。
- **导入/确认体验的稳定性补丁**：例如导入遮罩、统一确认弹窗、文本编码回退等，属于最终态产品体验增强。

### 5.2 可能的偏离（需要产品决策确认）

- 说明书默认“推演/问策/生辉/写作”在 v0 上会有更强的一体化工作台形态；当前实现存在“功能先跑通，UI 后对齐”的阶段性差异。
- “落笔域资产层”在说明书中是更系统的能力集合（模板/笔感/词典/锦囊等）；当前实现已很丰富，但“全局资产 vs 本书资产”的界面聚合方式可能与说明书最终态仍有差别。
