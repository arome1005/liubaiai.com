## 一、AI 写作核心难题与解决方案

### 1.1 长文一致性问题（最高优先级）

**问题根源**

当用户写到第 5 万字时，AI 对第 2 章的人物设定、已建立的世界规则、专有名词的写法已经"失忆"。圣经系统的静态注入只是第一步，真正的难点在于：如何让 AI 在任意章节的续写时，都能感知到"全书当前状态"而不只是"眼前这一段"。

**具体表现**

- 人物声音漂移：第 1 章冷酷的角色在第 8 章变得温柔
- 已死亡角色再次出现
- 圣经里定义的"灵力"在后续章节变成"元力""法力"交替出现
- 伏笔被遗忘，主线逻辑断裂

**解决方案：三层动态上下文架构**

```
┌─────────────────────────────────────────────────┐
│  Layer 1：静态真源（圣经）                         │
│  世界观 / 人物 / 术语 / 规则 → 永久注入，压缩后置顶  │
├─────────────────────────────────────────────────┤
│  Layer 2：动态摘要（滑动窗口）                     │
│  每写完一章自动生成摘要 → 存入章节元数据             │
│  AI 调用时按"最近 N 章摘要"动态拼入 context        │
├─────────────────────────────────────────────────┤
│  Layer 3：实体状态快照（Living State）             │
│  人物当前状态 / 物品位置 / 已发生事件 / 悬念清单      │
│  每次 AI 生成后，由另一次 AI 调用自动更新快照         │
└─────────────────────────────────────────────────┘
```

**落地方案**

1. **章节摘要自动队列**（步 20 扩展）：写完一章时自动触发后台摘要生成，不阻塞写作流。摘要约 200~400 字，重点提取：发生了什么、人物状态变化、引入了哪些新概念。
2. **实体状态追踪器**：在圣经模块新增"动态状态"子页，分为人物当前状态（心理/位置/关系）、物品/道具所在、悬而未决的冲突/伏笔三类。每次 AI 生成后，提示用户"是否更新状态"，或设置为自动更新（可关闭）。
3. **术语一致性检查**：圣经中的专有名词列表，在 AI 生成内容后自动扫描，标注可能存在的变体写法，供用户一键统一。

### 1.2 Context Window 分配博弈

**问题根源**

在一次侧栏 AI 调用中，多个内容源争夺有限的 context：


| 内容            | 估算 Token   |
| ------------- | ---------- |
| 圣经（完整）        | 3000~15000 |
| 当前章已有内容       | 2000~8000  |
| 最近 N 章摘要      | 1000~3000  |
| RAG 召回片段      | 500~2000   |
| 用户指令          | 100~500    |
| System Prompt | 500~1000   |


超出限额时，现有系统行为不可预测（静默截断），用户完全无感知，AI 质量突然下降。

**解决方案：Token Budget 可视化 + 优先级策略**

1. **侧栏顶部加入 Context 预算指示器**：用色块显示各项占比（灰=系统、蓝=圣经、绿=当前章、橙=摘要、红=RAG），接近上限时警告。
2. **明确优先级截断规则**（工程硬编码）：
  - 优先级：用户指令 > System Prompt > 实体快照 > 当前章 > 最近章摘要 > 圣经精简版 > RAG > 圣经完整版
  - 超限时从低优先级开始截断，不是整体丢弃
3. **圣经自动压缩**：圣经注入前，先用 AI 压缩为关键词+要点列表，而非全文注入。完整版只在用户明确要求时使用。

### 1.3 AI 生成内容与人类修改的融合问题

**问题根源**

用户接受 AI 生成的一段文字，修改了其中 60% 的内容，然后再次请求续写。AI 不知道哪些是它写的、哪些是人类改的，会在续写时"反向覆盖"用户的修改风格。

**解决方案：生成版本追踪**

1. **隐式版本标记**：AI 生成的段落在 CodeMirror 内部打上不可见标记（不影响纯文本导出），记录"生成时的摘要 hash"。
2. **下次续写时对比**：检测上次生成后编辑器内容是否有变化，若有，提示 AI "用户在上次生成的基础上做了修改，请延续修改后的风格而非原始生成风格"。
3. **风格锚定**：允许用户手动标记一段文字为"我的基准风格"，此后所有 AI 生成都以此为参照基准。

---

### 1.4 AI 幻觉在创作场景的特殊危害

**问题根源**

写作 AI 的幻觉不是"回答了错误的事实"，而是"生成了听起来完美但破坏了世界观的内容"。用户不会立刻发现，等发现时已经接受了大量后续内容，修改成本极高。

**解决方案：生成后一致性扫描**

1. **快速扫描模式**：每次生成结束后，在后台自动运行一次"一致性核查"，比对生成内容与圣经/实体快照，标注疑似矛盾点（用黄色高亮，不强制修改）。
2. **存疑标记**：AI 自身在生成时，可在 system prompt 中要求对"无法确认是否符合设定"的内容使用特定标记语（如`【待核查：此处涉及XXX设定】`），生成后自动转为 UI 中的存疑标注。
3. **一键对照**：侧栏提供"与圣经对照"按钮，高亮当前选中段落中出现的专有名词，并显示圣经中的定义。

---

### 1.5 推演分支的叙事爆炸

**问题根源**

三分支推演后，用户面对三条路线的内容，没有清晰的"选定 + 归档"工作流，大量推演内容既不丢弃也不进入正稿，成为沉没成本。

**解决方案：推演漏斗工作流**

```
推演生成 3 条路线
    ↓
用户在每条路线旁标记「倾向 / 放弃 / 保留备用」
    ↓
选定主线 → 一键「采纳入正稿」（转为正文草稿）
    ↓
其他路线 → 归入「灵感库」（流光模块）
    ↓
灵感库可按作品/章节检索，随时召回
```

新增"推演历史时间轴"：按章节显示所有推演记录，每次推演的三条路线以折叠卡片展示，选定路线有「已采纳」标记，放弃路线有「已存档」标记。

## 二、现有功能深化建议

### 2.1 圣经（落笔）系统

**当前状态**：圣经全链路、Prompt 模板、笔感样本、术语表注入已基本完成。

**可以更完善的点**

**① 圣经健康度评分**

在圣经页顶部显示一个"圣经完整度"进度条，按填写情况评分，提示用户还缺哪些关键信息（比如"您还没有添加主要冲突/核心矛盾"），对应到 AI 生成质量的实际影响。

**② 圣经字段智能建议**

用户新建作品时，输入类型（玄幻 / 言情 / 悬疑 / 科幻）后，自动推荐适合该类型的圣经字段结构。比如：

- 玄幻：修炼体系 / 门派 / 力量等级 / 禁忌
- 悬疑：时间线 / 关键证据 / 已知/未知 / 凶手思路
- 科幻：世界背景年代 / 技术边界 / 社会制度 / 专有术语

**③ 圣经内容版本历史**

圣经内容改动时保留历史版本，因为"中途修改了某个人物设定"是非常常见的场景，用户需要看到"我是从什么时候开始修改这个设定的"。

**④ 笔感样本的量化对比**

允许用户上传多段样本，AI 自动提取风格特征向量（句长分布、标点密度、常用句式），以可视化方式展示"我的风格 vs 目标样本"的差距。

---

### 2.2 RAG 系统（藏经）

**当前状态**：分块索引、阅读器、混合检索、PDF 已完成。

**可以更完善的点**

**① 检索结果的置信度展示**

当前 RAG 召回后直接注入 context，用户看不到召回了什么、为什么召回。建议在侧栏显示"本次参考了哪些片段"的折叠面板，标注相关度评分，让用户理解 AI 的信息来源。

**② 藏经内容的主动推送**

写作过程中，系统后台实时比对当前段落与藏经内容的语义相似度，如有高度相关的参考资料，在侧栏底部以卡片形式"浮现"推荐（类似编辑器的代码提示），用户点击可展开。不强制展示，可一键关闭。

**③ 藏经内容的结构化标注**

允许用户在阅读器中对藏经文档做分层标注：

- 蓝色：人物描写参考
- 绿色：场景/环境描写参考
- 橙色：对话风格参考
- 红色：情节结构参考

AI 在续写时可根据当前需要（描写/对话/情节）定向检索对应标注的片段。

**④ 跨作品藏经共享**

允许将某部作品的藏经设置为"共享"，其他作品也可检索该藏经。适合系列作品或同世界观的多个故事。

---

### 2.3 问策模块

**当前状态**：`/chat` 多轮对话 + 装配器分轨已完成。

**可以更完善的点**

**① 问策会话类型标签**

每条问策会话支持打标签（情节推演 / 人物分析 / 结构讨论 / 市场研究），方便按类型检索历史对话，解决"我上次跟 AI 讨论过这个问题但找不到了"的痛点。

**② 问策结论一键转圣经/正文**

问策对话产出了好想法（比如某个人物的动机分析），支持一键将 AI 的某条回复"转存"到圣经的指定字段，或转存为流光碎片。

**③ 问策 vs 推演的协同**

当用户在问策中得出了情节决策（比如"主角应该在第三章暴露身份"），支持一键"基于此结论开始推演"，将结论作为推演的起始条件自动填入。

## 三、留白页（首页 / 作品库）优化

留白页是用户每次打开 App 的第一眼，也是创作状态的"仪表盘"，目前偏向于"作品列表"的功能定位，可以做更多：

### 3.1 创作状态感知首页

**当前问题**：首页是作品卡片列表，用户进入后需要找到自己的作品、回忆上次写到哪、然后进入编辑器。"重新进入创作状态"的摩擦感高。

**改进方案：今日写作快速入口**

```
┌──────────────────────────────────────────────────┐
│  👋 欢迎回来，离你上次写作已过去 3 天               │
│  ──────────────────────────────────────────────  │
│  📖 继续《星渊纪》                                  │
│  上次写到：第七章 · 城墙之战（2126字）               │
│  [继续写作]    [先看一眼摘要]    [今日写作目标 ___字] │
└──────────────────────────────────────────────────┘
```

- **"先看一眼摘要"**：点击后展开上次写作内容的 AI 摘要（300 字以内），帮助用户快速回到创作心流，而不是干瞪着编辑器找状态。
- **今日写作目标**：用户可设置每日字数目标，首页显示进度，写作页顶部有实时字数进度条。

### 3.2 作品卡片信息密度提升

**当前**：卡片显示封面、标题、字数、进度。

**可以增加（可折叠，不强制）**：

- 上次写作时间（"3 天前"）
- 当前写到第几章/总章数
- 圣经完整度（小图标，绿=完整 黄=缺项 红=空）
- 当前情绪基调标签（紧张 / 舒缓 / 悬疑……由 AI 自动生成，也可手动设置）

### 3.3 创作数据面板

在作品库右侧或底部增加一个"创作数据"折叠面板：


| 指标      | 说明                 |
| ------- | ------------------ |
| 本周写作天数  | 连续打卡提示             |
| 最高产出日   | 帮助用户了解自己的创作节律      |
| 各作品字数占比 | 饼图，快速感知精力分配        |
| AI 使用频率 | 哪个模块用得最多（流光/推演/问策） |


这些数据不需要联网同步，本地 Dexie 即可支撑，但对提升用户粘性效果显著。

### 3.4 灵感快录入口

留白页右下角增加一个悬浮的"快记"按钮，点击后弹出一个极简的输入框（不需要选择作品、不需要选章节），输入后存入全局流光池，后续再整理到具体作品。适合用户有想法但没在写作状态时的随手记录。

### 3.5 作品健康警示

如果一部作品超过 30 天没有更新，作品卡片边缘出现淡淡的提醒色，悬停时提示"您已有 30 天未写作，是否需要 AI 帮您回顾上次进度？"。温和、非打扰式。

## 四、沉浸式写作页优化

沉浸式写作是用户停留时间最长的页面，也是"创作心流"的核心载体，任何干扰都是代价。

### 4.1 当前存在的体验断裂点

1. 用户从 Hub 进入编辑器，没有"进入状态"的过渡感
2. 侧栏开关会引起布局抖动，打断视线
3. 顶栏功能密集，沉浸感不足
4. 没有"当前写作上下文"的随时可查摘要

### 4.2 进入沉浸态的过渡体验

**① 写作前的"热身"提示**（可关闭）

进入编辑器时，如果超过 3 天未写作，显示一个轻量的"上次写到哪"浮层（3秒自动消失，或按任意键关闭）：

```
上次停笔处：
"——城墙轰然倒塌，沈煜站在废墟中央，看着……"

本章已写：1,247 字    全书进度：34%
[开始写作]
```

**② 沉浸态的信息层级**

沉浸态下，所有非核心 UI 收入两个层级：

- **Level 0（默认）**：只有编辑区，顶栏缩为高度极小的指示条（仅显示章节名+字数）
- **Level 1（鼠标移至顶部时）**：展开完整顶栏
- **Level 2（⌘+K 或右侧轻触）**：展开侧栏 AI

### 4.3 写作过程中的辅助信息

**① 实时字数与节奏感知**

在编辑器右下角显示：

- 当前会话字数（本次打开后写了多少）
- 全章字数
- 预计阅读时长
- 今日目标进度（如已设置）

**② 句子长度可视化（可选功能）**

在段落左侧显示极细的颜色条，代表该段落的平均句长（蓝=短句/紧张节奏，绿=中等，橙=长句/舒缓叙事）。帮助用户感知自己的写作节律是否单调。这是一个对有意识提升写作能力的用户很有价值的功能。

**③ 写作中的浮动建议（非强制）**

当用户停笔超过 3 分钟时，侧栏出现一个轻量提示卡片（不遮挡编辑区）：

```
您已停笔 3 分钟
─────────────────
💡 继续上一段的情绪线？
💡 加入一个场景转换？
💡 让角色说出他真正想说的话？
[不了，让我再想想]  [生成一个开头让我参考]
```

三个提示由 AI 基于当前上下文生成，每次不同。按 Esc 或继续打字自动消失。

### 4.4 沉浸态中的快捷操作设计


| 快捷键             | 功能                     |
| --------------- | ---------------------- |
| `⌘ + .`         | 切换沉浸 / 标准模式            |
| `⌘ + /`         | 在光标位置插入分隔符（场景转换）       |
| `⌘ + '`         | 快速记录灵感到流光（不离开编辑器）      |
| `⌘ + ↑/↓`       | 在章节间切换（不需要回到侧边栏）       |
| `双击空白处`         | 隐藏/显示段落节奏色条            |
| `⌘ + Shift + F` | 聚焦模式：只高亮当前段落，其他段落降低透明度 |


### 4.5 "当前上下文"随时可查

在侧栏 AI 顶部固定一个"本章上下文"折叠卡：

```
▸ 当前章节摘要（AI 自动生成）
▸ 本章已引入的人物
▸ 圣经中与本章高度相关的条目
▸ 本章未解决的冲突/悬念
```

这样用户不需要离开写作页去查圣经，降低上下文切换成本。

### 4.6 写作完成后的"收笔"体验

完成一章后，点击"完成本章"触发一个简单的收笔流程：

1. AI 自动生成本章摘要（200字，可编辑）
2. 提示用户更新实体状态快照（人物/物品/事件）
3. 显示"本章写作统计"（用时/字数/AI 辅助次数）
4. 一句鼓励（不矫情，可关闭）

## 五、侧栏 AI 优化

### 5.1 多轮对话记忆

**当前问题**：侧栏 AI 每次调用是否携带上轮对话历史？如果不是，用户需要在每次请求中重复背景。

**改进**：侧栏 AI 保持一个"本章写作会话"，在用户关闭侧栏前保持上下文。用户可以像跟编辑对话一样说"不，让它更暗一点"——AI 理解"它"是指上一次生成的段落。

### 5.2 生成结果的精细操作

当前 diff 合并是二元的（接受/拒绝）。增加：

- **段落级接受**：长段生成时，可以只接受某几段
- **词语替换**：双击生成文本中的某个词，出现同义词替换建议
- **调整力度**：生成结果下方有滑块"再强烈一点 ←→ 再克制一点"，点击后在原有生成基础上微调，不重新生成全部

### 5.3 装配器的可视化

装配器（圣经+材料+草稿区）目前以文字配置为主。增加：

- **注入预览**：装配器底部显示"预计注入 Token 数"和各组件占比，让用户了解 AI 拿到的信息量
- **快速预设**：提供几个常用装配预设（轻量/标准/全量），一键切换，不需要每次手动选择

### 5.4 AI 调用历史

侧栏 AI 底部增加"本章 AI 记录"折叠面板，显示本章所有 AI 调用的简要记录（时间/操作类型/使用 Token），方便用户了解 AI 使用模式，也方便调试。

## 六、各模块深化

### 6.1 流光（碎片写作）

**① 碎片的自动归类**

流光碎片目前依赖用户手动整理。增加 AI 自动归类建议：每次新增碎片后，AI 在后台分析其内容，建议"这个碎片可能属于《星渊纪》第四章的情节线"，用户一键确认或忽略。

**② 碎片的拼接功能**

选中多个碎片，点击"拼接"，AI 尝试将这些碎片组合成一段连贯的文字，保留原文的意象和风格。适合灵感碎片整合成正式段落的场景。

**③ 碎片与正稿的桥接**

在编辑器中选中一段文字，右键"转为流光碎片"，同时在碎片上记录"来自第X章"，双向关联。

### 6.2 推演模块

**① 推演分支的"血缘图"**

如果用户在一个分支上再次推演，形成多级分支，以树状图展示推演历史，每个节点可点击查看内容。

**② 推演导出**

支持将选定的推演路线一键导出为：

- 正文草稿（进入编辑器）
- 大纲要点（精简版，进入圣经的故事线字段）
- 流光碎片池（分段打散）

**③ 基于人物视角的推演**

除了情节分支，增加"人物视角推演"：同一场景，分别从 A / B / C 人物的视角生成独白或行动选择，帮助作者理解各角色的动机与立场。

### 6.3 生辉（仿写）

**① 风格解析报告**

仿写前，先对目标样本生成"风格解析报告"，以可视化方式展示：

- 平均句长
- 常见句式结构
- 词汇丰富度
- 情绪曲线（段落情绪起伏）
- 典型修辞手法

用户理解了"在学什么"后，仿写练习的目的感更强。

**② 仿写练习与对比**

用户写一段，AI 分析与目标风格的差距，给出具体的"你用了长句，目标风格倾向短句+排比"等改进建议，而不只是生成一段让用户看。

**③ 版权边界提示**

在生辉入口醒目位置注明：本功能用于风格学习，建议使用自己拥有版权的样本，或已授权的公版文学作品。

## 七、关联 App 与生态集成

### 7.1 输入端集成（想法从哪来）

**① Apple Notes / 备忘录**

许多创作者习惯在手机备忘录里记录灵感。接入 iCloud Notes 读取权限，支持将指定标签（如"写作灵感"）的备忘录自动同步到流光碎片池。

技术路径：通过 macOS/iOS App 的 Share Extension 实现，或通过 URL Scheme 接收分享。

**② Readwise / 微信读书 / Kindle**

创作者在阅读时划的重点往往是最好的藏经素材。支持导入 Readwise 导出格式（CSV/Markdown），自动分类进藏经。

**③ 录音转写（语音灵感）**

手机端支持按住录音，自动转写为文字存入流光。或接入 iOS 的 Shortcuts，实现"说完话直接存入留白"。

### 7.2 写作过程中的参考集成

**① 维基百科 / 百度百科快查**

在编辑器中选中一个专有名词，右键"查询资料"，在侧边栏显示简短的百科摘要（不离开写作页）。适合历史小说、科幻小说等需要大量查资料的类型。

**② 地图集成（场景地理）**

对于历史小说、冒险小说，场景的地理逻辑很重要。支持在圣经中嵌入一个轻量的地图标注工具（可以是手绘地图上传，也可以是 Google Maps 标注），AI 续写时可参考地理关系。

**③ 时间轴可视化**

推演和正文的时间线整合到一个可视化时间轴视图，支持多条线并行（主线/副线/回忆），类似 Scrivener 的 cork board 但以 AI 辅助为核心。

### 7.3 输出端集成（写完往哪去）

**① 稿件格式导出**

支持导出为：

- Word .docx（投稿编辑常用）
- TXT / Markdown（通用）
- 番茄小说 / 晋江 / 起点的发布格式（对标国内连载平台）
- EPUB（自出版）

每种格式有预设模板（字体/行距/章节样式），一键导出符合平台要求的格式。

**② 语雀 / Notion / Obsidian 同步**

作为"第二大脑"工具的使用者越来越多，支持将圣经内容双向同步到 Notion / Obsidian，让创作者的知识管理体系打通。

技术路径：Notion API（官方支持）、Obsidian Vault 文件夹映射（通过本地文件读写）。

**③ 稿件版本管理（Git-like）**

每次保存时自动创建版本快照，支持任意版本对比（类似 Git diff，但以章节为单位展示差异），并可回滚到任意历史版本。不需要用户理解 Git，只需要"时间轴 + 回到这个版本"的概念。

### 7.4 创作者社区集成（中长期）

**① 私密读者模式**

允许用户生成一个"私密阅读链接"，分享给编辑或信任的读者，读者可以在特定章节留下评论（类似 Google Docs 的评论），作者在 App 内查看。

**② 写作伙伴功能**

两位用户可以互相"订阅"对方的创作进度（仅字数/更新频率，不看内容），互相鼓励打卡。

## 附录：一句话总结各难题的根本解法


| 难题         | 根本解法                               |
| ---------- | ---------------------------------- |
| 长文一致性      | 不靠单次注入，靠动态三层架构（静态圣经 + 滑动摘要 + 实体快照） |
| Context 溢出 | 定义优先级截断规则，可视化 Token 预算，圣经自动压缩      |
| AI 幻觉      | 生成后自动一致性扫描 + 存疑标记，让用户有感知           |
| 分支爆炸       | 推演漏斗工作流：选定→采纳正稿，放弃→归档流光            |
| 写作心流断裂     | 重入摘要、停笔提示、沉浸态信息层级、全键盘操作            |
| 生成与修改融合    | 生成版本隐式追踪 + 风格锚定，让 AI 感知人类的修改方向     |
