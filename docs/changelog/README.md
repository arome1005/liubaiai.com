# 变更记录 · 供复查（Claude / 人工）

本目录用于**按对话/子功能**沉淀「做了什么、怎么做的、哪些文件」，方便你**整包丢给克劳德**或自己过一遍，而不用翻聊天记录。

## 和 git 的关系

- **不替代** `git log` / PR 描述；这里偏**产品意图 + 落点 + 验证点**。
- 大重构可只写「见 PR / commit xxx」并链到仓库。

## 新写一条时怎么做

1. 复制 `_template.md` 为 `YYYY-MM-DD-短英文主题.md`（同目录下）。
2. 按模板填全；**相关文件**用表格最省事。
3. 在本文件下方的 **索引** 里加一行（日期、标题、链接）。

**建议写清楚的几件事（复查最爱问）**


| 块        | 写什么                                                 |
| -------- | --------------------------------------------------- |
| 目标 / 非目标 | 用户要什么；明确不做什么                                        |
| 背景       | 改之前代码里**已经**有什么、缺什么（避免重复造轮子说明）                      |
| 落点       | 为什么放在 `hook` / `components/某域` / `util` / `ai`（可一句） |
| 未改动      | 尤其「未动 `EditorPage` / `AiPanel`」等约束时                 |
| 验证       | 手测步骤或已跑 `tsc` / 测试命令                                |


**子页面 / 大文件约定（与 `.cursor/rules` 一致时可在条目中写一句）**

- 推演：`V0TuiyanPage.tsx`、 `src/components/tuiyan/`*、`useTuiyan*.ts`
- 写作编辑器：`EditorPage.tsx`、`AiPanel.tsx`、`src/components/ai-panel/`*（避免单文件继续膨胀时优先抽 hook/子组件）
