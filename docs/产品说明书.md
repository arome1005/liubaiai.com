## 一、产品定位与核心理念

**留白写作**是一款面向长篇虚构创作（网文、小说、剧本等）的 **AI 辅助写作平台**。

### 核心原则


| 原则         | 落地方式                                                             |
| ---------- | ---------------------------------------------------------------- |
| **数据主权**   | 本地优先（IndexedDB）；云同步可选（Supabase）；参考库（藏经）**始终本地**不上云；随时可导出 ZIP 备份。 |
| **AI 可解释** | 每次 AI 调用展示「本次使用材料」摘要；门控与成本提示在前；生成结果进草稿区，用户确认后再合并。                |
| **成本可控**   | 本地粗估 token；超阈值数字确认；本会话 / 本机累计展示；高危操作二次确认。                        |
| **心流优先**   | 沉浸写作模式；全局快捷键灵感速记；七模块平铺导航减少迷路。                                    |


### 七大模块一览


| #   | 模块名    | 路由                      | 一句话                                                                       |
| --- | ------ | ----------------------- | ------------------------------------------------------------------------- |
| 1   | **留白** | `/library`              | 作品库门户：新建、管理、进入各作品                                                         |
| 2   | **推演** | `/logic`                | 大纲规划、一致性扫描、剧情三分支                                                          |
| 3   | **流光** | `/inspiration`          | 灵感碎片捕获、AI 扩容、转入正文                                                         |
| 4   | **问策** | `/chat`                 | 策略对话咨询，承接推演分支跟进                                                           |
| 5   | **落笔** | `/work/:id/bible`（锦囊子页） | 七模块之一、§5 能力总域；**锦囊**仅为原「圣经」UI 名。**正文编辑** 为 **「写作」** `/work/:id`（新建作品默认进入） |

## 二、账号与数据架构

### 2.1 注册与登录

- **邮箱 OTP 注册**：后端发送验证码 → 前端验证 → Supabase Auth 建号 → `signInWithPassword`。
- **登录**：邮箱 + 密码；支持「忘记密码」邮件重置。
- **会话**：Supabase JWT；前端通过 `@supabase/supabase-js` 管理。
- **未登录保护**：所有业务页（`/library`、`/logic` 等）需登录；未登录自动跳转 `/login`。
- **演示包**：新注册用户首次进入自动注入五部示例书 + 流光碎片，标签「演示包」可删。

### 2.2 存储双轨


| 场景                        | 存储方式                                                                   |
| ------------------------- | ---------------------------------------------------------------------- |
| 未配 Supabase 环境变量          | **纯 IndexedDB**（Dexie），全部数据本地                                          |
| 配齐 `VITE_SUPABASE_`* 且已登录 | **Hybrid**：写作/本书锦囊/风格卡/流光/问策 → Supabase（RLS）；**参考库（藏经）→ 仍在 IndexedDB** |
| 登出或未配                     | 自动回退纯 IndexedDB                                                        |


### 2.3 备份与恢复

- **全量导出**：ZIP 包含 `data.json`（所有写作数据）+ 参考库。
- **全量恢复**：替换当前用户所有数据（**危险操作**，二次确认）。
- **合并导入**：ID 映射后追加，不覆盖已有。
- **备份软提醒**：约 30 天未导出时，设置页出现提示条。

## 三、全局壳层与导航

### 3.1 顶栏（AppShell）

适用于除写作页外的全部 Hub 页面。


| 区域              | 内容                                                                                           |
| --------------- | -------------------------------------------------------------------------------------------- |
| **品牌区**         | 「留」字渐变方标 + 「留白写作 BETA」；点击回首页。                                                                |
| **作品上下文**（书本图标） | 当 `lastWorkId` 存在时展示：当前作品标题/章/字数/标签；快捷去「继续编辑」「查看大纲」「切换作品」。加载态为 Skeleton，作品不存在时为错误提示 + 返回作品库。 |
| **七模块导航**       | 留白、推演、流光、问策、落笔、生辉、藏经；**全部露出**，不用「更多」收纳；激活态高亮 + 底部短条；窄屏可横向滚动。「落笔」无当前作品时为浅灰色。                  |
| **搜索按钮**        | 打开命令面板（`⌘/Ctrl+K`）。                                                                          |
| **通知铃**         | 占位，暂未接入业务。                                                                                   |
| **用户菜单**        | 头像/邮箱、登出。                                                                                    |
| **设置齿轮**        | 链至 `/settings`。                                                                              |


### 3.2 命令面板（GlobalCommandPalette）

- `⌘/Ctrl+K` 唤起（正文编辑区内不抢键）。
- 搜索作品名、标签、章节标题；跳转到对应模块。
- `Esc` 关闭，`↑↓` 选择，`Enter` 打开。

### 3.3 写作壳（EditorShell）

写作页 `/work/:id` 使用独立壳层，不显示七模块导航：


| 区域       | 内容                                           |
| -------- | -------------------------------------------- |
| **左**    | 「作品库」返回按钮 + 作品/章节标题                          |
| **中**    | 编辑器注入的面包屑/保存态                                |
| **右**    | 搜索、用户菜单、「打开/关闭右栏」                            |
| **右侧栏**  | AI / 概要 / 本书锦囊 / 参考 四个 Tab                   |
| **沉浸模式** | `Alt+Z` 或 URL `?zen=1`；隐藏顶栏，右上角退出按钮；`Esc` 退出 |


### 3.4 作品子导航

在 AppShell 下访问 `/work/:id/summary` 或 `/work/:id/bible` 时，顶栏下方出现：

- 作品名 + 字数
- 写作 / 概要 / 锦囊 / 作品库

### 3.5 首次 AI 门控

- 首次调用任何 AI 功能前，弹出「虚构创作声明」确认。
- 勾选「已阅读」+ 点击「开始使用」后解锁。
- 全站统一：写作侧栏、推演、流光、问策、生辉均受此门控。

## 四、留白 — 作品库门户

### 路由：`/library`

### 4.1 功能清单


| 功能       | 说明                                                    |
| -------- | ----------------------------------------------------- |
| **作品列表** | 网格 / 列表视图切换；按更新时间或标题排序；搜索（标题 + 标签）。                   |
| **统计区**  | 作品总数、有章节的作品数、总字数（万）、章节总数。                             |
| **作品卡片** | 封面图（可上传，≤400KB data URL）/ 色块占位；标题、标签 chip、字数、章数、更新时间。 |
| **卡片操作** | 点击进入写作页；悬浮显示「继续写作」「推演」「锦囊」快捷入口；编辑元数据；设置封面；删除。         |
| **新建作品** | 打开 v2 表单弹窗（`WorkFormModal`）。                          |
| **导入作品** | 支持 `.txt` / `.md` / `.markdown` / `.docx`；自动按章回标题切章。  |
| **空态**   | 「开始你的创作之旅」引导 + 新建按钮。                                  |
| **加载态**  | 页眉 + 「加载中…」文案。                                        |


### 4.2 新建作品弹窗（WorkFormModal）


| 字段         | 说明                                  |
| ---------- | ----------------------------------- |
| **作品名称** * | 必填                                  |
| **作品简介**   | 可选自由文本                              |
| **作品状态**   | 连载中 / 已完结 / 归档（chip 单选）             |
| **作品标签**   | 平台定位、题材类型、创作类型三组预设 chip 可多选；底部自定义输入 |
| **同人提示**   | 选中「同人」或「同人衍生」时出现版权提示                |
| **AI 提示**  | 「标签将影响 AI 侧栏中的风格与节奏提示」              |


创建成功后：`navigate('/work/${id}?zen=1')` → 进入沉浸式写作。

### 4.3 数据模型

- **Work**：`id, title, description, status, coverImage, tags, createdAt, updatedAt, progressCursor`
- 存储：IndexedDB `works` 表 / Supabase `work` 表

### 4.4 关联模块


| 关联   | 说明                                   |
| ---- | ------------------------------------ |
| → 落笔 | 「继续写作」进入 `/work/:id`                 |
| → 推演 | 卡片快捷入口                               |
| → 锦囊 | 卡片快捷入口                               |
| → 导入 | `importWorkFromFile` → 章节切分 → 自动建卷建章 |

## 五、推演 — 大纲与逻辑规划

### 路由：`/logic`

### 5.1 功能清单


| 功能           | 说明                                                        |
| ------------ | --------------------------------------------------------- |
| **作品/章选择**   | 下拉选择当前作品与目标章节（支持从路由 `state.preferWorkId` 预选）。             |
| **规则一致性扫描**  | 比对正文 vs 锦囊禁写、风格卡禁用套话、术语表、人物禁忌；结果为告警卡片列表。                  |
| **LLM 语义扫描** | 调用 AI 分析逻辑矛盾；可选范围（本章 / 最近 5 章 / 全书）；可取消；结果可忽略/恢复。         |
| **三分支预测**    | 输入倾向提示 → AI 生成三条剧情走向（标题 + 摘要）；可 **写入写作侧栏草稿**、可 **去问策跟进**。 |
| **时间线**      | 展示锦囊时间线事件列表；可快速追加新事件（标签/备注/可选关联章）。                        |
| **世界观速览**    | 只读展示世界观条目。                                                |
| **地图与地点事件**  | SVG 地图视图 + 地点列表 + 事件列表；新增地点（名称/备注/坐标）、新增事件（可关联章节）。        |
| **三态**       | 无作品 / 无章节 / 无结果时有引导文案。                                    |


### 5.2 数据模型

- 读取：`Work`、`Chapter`、`ChapterBible`、`WorkStyleCard`、`BibleCharacter`、`BibleWorldEntry`、`BibleTimelineEvent`、`BibleGlossaryTerm`、`LogicPlaceNode`、`LogicPlaceEvent`
- 写入：`LogicPlaceNode`、`LogicPlaceEvent`、`BibleTimelineEvent`（快速追加）
- AI 草稿：写入 `sessionStorage` 的 `liubai:aiPanelDraft:v1:{workId}:{chapterId}`

### 5.3 关联模块


| 关联         | 数据流                                                             |
| ---------- | --------------------------------------------------------------- |
| → **写作侧栏** | 三分支「写入草稿」→ `writeAiPanelDraft` → 写作页 AiPanel 显示 → 合并入正文         |
| → **问策**   | 三分支「去问策跟进」→ `navigate('/chat', { state: { prefill } })` → 新会话预填 |
| ← **本书锦囊** | 读取人物/世界观/时间线/术语/禁忌                                              |
| ← **风格卡**  | 三分支 AI 调用注入风格卡 + 标签侧写                                           |

## 六、流光 — 灵感碎片管理

### 路由：`/inspiration` + 全局弹层

### 6.1 功能清单


| 功能          | 说明                                                          |
| ----------- | ----------------------------------------------------------- |
| **碎片列表**    | 时间、正文摘要、标签、归属作品、所属集合；网格/列表视图。                               |

- **InspirationFragment**：`id, workId?, collectionId?, body, tags[], createdAt, updatedAt`

### 6.3 关联模块


| 关联         | 数据流                         |
| ---------- | --------------------------- |

## 八、本书正文编辑（写作页）

### 路由：`/work/:id`（EditorShell）

### 8.1 功能清单


| 功能        | 说明                                                |
| --------- | ------------------------------------------------- |
| **章节树**   | 左侧章节列表，支持新建、切换（不残留上章正文）、排序（拖拽重排）、删除。              |
| **正文编辑**  | CodeMirror 6；自动保存（防抖）；字数统计；行尾模式。                  |
| **保存与冲突** | `updatedAt` 乐观锁；冲突时顶栏提示「重新载入本章」。                  |
| **草稿恢复**  | `draftRecovery`：每章内容防抖写入 localStorage，异常关闭后重进可恢复。 |
| **查找替换**  | 全书搜索（`searchWork`）+ 本章查找替换。                       |
| **导出**    | 单章 TXT/DOCX、全书 TXT/DOCX/MD。                       |
| **快照**    | 手动快照 + 自动快照（保存时写入）；每章最多 50 条、超 7 天清理。             |
| **卷管理**   | 新建卷、卷标题编辑、删除（需先删完卷内章节）。                           |
| **续写**    | 顶栏「续写」→ 打开 AI 侧栏续写模式并自动请求。                        |
| **抽卡**    | 「抽卡」模式：概要作大纲，不加额外要求。                              |
| **编辑器宽度** | 自定义（拖拽调整 720–1600px）或自适应。                         |
| **排版**    | 字体族、行高、稿纸底色（`data-paper-tint`）。                   |
| **沉浸写作**  | `Alt+Z` 或 `?zen=1`；退出 `Esc`。                      |


### 8.2 AI 侧栏（AiPanel）


| 功能           | 说明                                                             |
| ------------ | -------------------------------------------------------------- |
| **模型选择**     | 支持 OpenAI / Anthropic / Gemini / 豆包 / 智谱 / Kimi / 小米 / Ollama。 |
| **运行模式**     | 续写 / 扩写 / 润色 / 自定义 / 抽卡。                                       |
| **额外要求**     | 自由文本；可从提示词模板库填入。                                               |
| **材料勾选**     | 邻章概要 / 本章锦囊字段 / 全书锦囊板块 / RAG。                                  |
| **「本次使用材料」** | 可折叠摘要：模型、作品/章、风格卡、本书锦囊、摘录、标签、RAG、粗估 token。                     |
| **生成**       | 流式输出到 **草稿区**（独立于正文）。                                          |
| **草稿合并**     | 打开 diff 弹窗（`AiDraftMergeDialog`）：行级对比、追加到章末 / 光标位插入 / 替换选区。    |
| **成本控制**     | 粗估 token；超阈值（默认 20 万）数字确认 + 最短停留 1.2s + 冷却 1.5s；本会话/本机累计展示。    |
| **调性提示**     | 禁用套话命中 + 句长对比（仅参考，不阻断）。                                        |
| **降级重试**     | 上下文类错误 → 「精简并重试」自动缩短注入。                                        |


### 8.3 右栏四 Tab


| Tab            | 内容                       |
| -------------- | ------------------------ |
| **AI**         | AiPanel 完整面板             |

## 十三、跨模块联动总览

以下是产品核心的 **四条联动链路**（终验收必测）：

### 链路 1：推演 → 写作

```
推演页：生成三分支 → 点击「写入草稿」
  → sessionStorage 写入 liubai:aiPanelDraft
  → 用户切到写作页（/work/:id）
  → AiPanel 读取草稿展示
  → 用户点击「合并入正文」→ diff 弹窗 → 确认
  → 正文更新
```

### 链路 2：推演 → 问策

```
推演页：三分支结果 → 点击「去问策跟进」
  → navigate('/chat', { state: { prefill: 分支内容 } })
  → 问策页创建新会话，user 消息预填分支内容
  → 用户可直接发送继续讨论
```

### 链路 3：流光 → 写作

```
全局速记弹层（Alt+S）→ 保存碎片
  → 可选：去流光扩容 → 生成五段 → 保存为新碎片
  → 流光页：选碎片 → 转入章节
    - 方式 A：章末追加（分隔线 + 碎片文本）
    - 方式 B：光标位插入（sessionStorage handoff → 写作页执行）
    - 方式 C：侧栏草稿合并
  → 保存 → 重进写作页，内容仍在
```

### 链路 4：藏经 → 写作 AI

```
藏经页：导入参考书 → 分块 + 索引
  → 写作页右栏「参考」Tab：开启 RAG + 勾选参考书
  → 用户触发 AI 生成
  → 装配器 (assemble-context) 执行 top-k 检索
  → 命中条目注入 AI 上下文
  → 「本次使用材料」摘要展示命中信息
```
