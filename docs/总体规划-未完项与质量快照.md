### 3.2 全仓 ESLint / 规则（`master-checklist` §D.4）

以下为 **已知未清光** 的类型（`npm run lint` 仍可能报错/告警），策略是 **触达文件时顺带收**，不单开大重构 PR：

- `**react-hooks/exhaustive-deps`（warning）**：如 `AiPanel.tsx`、`EditorPage.tsx` 等。  
- `**react-refresh/only-export-components`**：`CodeMirrorEditor.tsx`、`EditorZenContext.tsx` 等（同文件导出常量/非组件）。  
- **登录页 `LoginHero` / BlobCharacter**：`react-hooks/refs` 相关，已有 `eslint-disable` 块标明意图。  
- `**EditorShell` / `AppShell`**：`queueMicrotask` 等与规则适配相关。  
- **Vite build 体积**：主 chunk >500kB 提示；与性能/路由级 `import()` 合并规划。

## 四、新开 AI 对话时建议附上的「话术模板」

把下面整段或按需删减贴到新窗口 **第一条消息**，可减少歧义、降低来回追问。（可按你当前任务改「目标页」「不要做」两栏。）

---

### 模板 A：通用（接本仓库总体规划）

```text
我在做「留白写作」仓库（Vite/React/TS，主清单 design/master-checklist.md）。
请先读 docs/总体规划-未完项与质量快照.md，再读 design/master-checklist.md 的 §E 与 §B 未划线项。

【本期目标】：（例如：只做 §E.2.3 写作壳 v0 级验收，不改业务逻辑）
【不要做】：（例如：不接生产 Supabase、不改可选步 13 真流式、不大范围改 ESLint）
【验收】：npm run build；涉及页手测 smoke + 空态/长列表目检；与 v0 对照 design/v0-ui-reference/docs/ui-design-specification.md

改完后请：更新 master-checklist 对应 ~~ 划线、同步更新 docs/总体规划-未完项与质量快照.md 的质量快照表（若完成一整块 §E.2.x）。
```

### 模板 B：只修 bug / 技术债

```text
目标：修复（或缩小范围处理）docs/总体规划-未完项与质量快照.md §三 中列出的项：___（填文件名或规则名）。
约束：不扩大功能范围；改完 npm run build；触达文件顺带修 eslint 同类问题即可，不必全仓 lint 清零。
```

### 模板 C：只推进 UI 相似度（§E 某一项）

```text
按 design/master-checklist.md §E.2.x 的「整页 v0 级验收」推进：本步是 E.2.__（填数字），涉及页面：___。
参考：design/v0-ui-reference（禁止 import 子工程源码）、主应用已有 src/components/ui/* primitive。
完成标准：该页与 ui-design-specification.md 极高相似（约 96%～98%），功能不退化；然后划掉 master-checklist 对应行并更新 docs/总体规划-未完项与质量快照.md。
```

### 模板 D：让 AI **更准确理解你意图**时可加上的短句

- 「**真源顺序**：先 `master-checklist`，再 `docs/总体规划-未完项与质量快照.md`，代码以 `src/` 为准。」  
- 「**本期不碰**：云端/生产部署、Hybrid 远端推送、可选 backlog。」  
- 「**完成定义**：能勾选 master-checklist 对应行 + 修订记录一行；不需要写长篇新文档。」  
- 「**若不确定**：先列出你将改的文件路径再动手。」  
- 「**与 v0 差异可接受项**：字体、CodeMirror 接缝、极端长标题——按总体规划 §E 说明。」
